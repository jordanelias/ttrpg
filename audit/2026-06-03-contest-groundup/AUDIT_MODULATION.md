# Modulation Model — Audit (NERS all directions + generality)

`[SELF-AUTHORED — bias risk]` Diagnostic deployed on my own uncommitted proposal; attacked, not confirmed. Verdict first, worst first.

## Verdict
The modulation **works** (context/adjudicator/tension/F1 all demonstrated, 31/31 tests) but is **not NERS-compliant yet** and **not general enough** for the stated goal — *any* two-party adjudicated discourse. Two structural gaps dominate: it hard-codes things a general engine must make configurable, and it carries an inert second channel.

## Phase 0 — decomposition
dice (one reception roll per move) + heavy deterministic modulation (resonance, readiness, leverage, self-gating, defeat-conditions) + continuous resources (standing, room) + clock (merits). The stochastic surface stays small; correct composite.

## NERS — all directions
- **N — fail (over-build).** Standing/room carry two mechanical roles: they feed **readiness** (the verdict multiplier) *and* **reception leverage** (the roll). The stress toggle shows the leverage role is nearly inert — readiness alone reproduces the build payoff (BTC 0.72 vs 0.71), while readiness-off collapses it (0.04). Lesson 1: collapse to one role — standing/room drive readiness; the reception roll keeps only faculty + on-ground. Resonance and readiness *themselves* are necessary (each drives a required, non-substitutable behavior — context/adjudicator modulation, and build-then-close).
- **R — fail at the extremes.** Resolution collapses at the floor (faculty 1 → 90% draws) and degrades high (f7 → 40%): merit-per-hit doesn't scale with pool, so the clock is mis-depthed away from mid faculty. The matched-appeal decisiveness is also sharp at strong role/character gaps (~0.94). Symmetry itself holds at all scales.
- **S — partial.** New *contexts* are config-only (the appeal venue worked with zero code — genuinely smooth). But cross-mode smoothness is unvalidated (the four sub-systems are scaffolds), and there is a single win-condition shape, so modes whose decision rules genuinely differ are not yet smoothly expressible.
- **E — strained.** The model has grown; the inert leverage-coupling is pure overhead, and the double resource-coupling is not transparent. The strategic story ("match the appeal, build before you close, exploit the role/character gap") is intuitable; collapsing N restores elegance.

## Generality — "any two-party adjudicated discourse" (the explicit requirement)
**Already configurable:** the discursive context (role weights), the adjudicator's character + discipline, the stasis grounds, thresholds, defeat-condition counts. New contexts need no code (demonstrated).

**Hard-coded — the blocks to full generality:**
1. **The appeal axis is fixed to three proofs (ethos/pathos/logos).** A framework with different argument categories (e.g. scripture/reason/precedent in a disputation) cannot be expressed. → parameterize the proof set (N types, each with a resonance profile and an optional resource/target).
2. **The win condition is one shape (merits threshold OR clinch).** Vote tallies, graded proof-hierarchy verdicts, grace-granting, consensus are not expressible. → make the win-condition pluggable.
3. **The adjudicator is singular — but the brief says "adjudicator(s)."** Panels (jury, bench, council) with per-member character and an aggregated verdict are unsupported. → support an adjudicator panel that aggregates resonance/verdict.
4. **The defeat-condition catalogue is fixed in type** (barred/contradiction/evasion/silence). → make the fault set configurable.

The engine is a sound *core* for two-party adjudicated discourse and already generalizes the context and the adjudicator's disposition; it does not yet generalize the **proof axis**, the **win condition**, the **adjudicator multiplicity**, or the **fault set** — which is what "any framework" requires.

## Remediation (worst-first; lesson-mapped)
1. **Generality (the requirement) — make four things configuration, not code:** a parameterized proof/appeal set; a pluggable win-condition; an adjudicator **panel** (aggregate over members) per "adjudicator(s)"; a configurable fault catalogue. This is the structural work; the resonance/readiness/clinch machinery already supports it once these are abstracted. *(Surfaced lesson: the skill's six lessons don't cover **generality/parameterization** — flagged rather than forced.)*
2. **N / Lesson 1 — collapse the standing/room double-role:** readiness only; drop the inert resource→reception-leverage coupling. Re-test shows no behavior change (0.72→0.71). Quick, simplifying.
3. **R — depth across faculty:** scale threshold (or merit) with pool so the floor and high faculty resolve (currently 90%/40% draws). Balance pass.
4. **R — decisiveness:** soften the sharp matched end via reception variance or flatter weights. Feel call.

## Stage-4 re-test (proposed fixes)
- Fix 2 (drop leverage-coupling) is **validated** by the toggle probe — readiness alone reproduces the dynamics, no new defect.
- Fixes 1, 3, 4 require implementation; re-run the audit + stress after.

## Bottom line
Sound dynamics, real over-build (one inert channel), mis-depthed clock at the extremes, and — the headline against the brief — a core that generalizes context and adjudicator-character but **hard-codes the proof axis, the win condition, the adjudicator multiplicity, and the fault set**. Meeting "any two-party adjudicated discourse" means abstracting those four. Recommend doing that (plus the N collapse) before ratifying.

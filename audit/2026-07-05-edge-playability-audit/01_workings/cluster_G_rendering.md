## Notes where the verdict isn't obvious

**The entire `scene_outcome` family is structurally absent from §3.1.** Re-reading the 10-row table line by line: none of `contest_resolved`, `battle_concluded`, `investigation_resolved`, `combat_resolved` appear. Only `combat_resolved` gets an unintegrated aspirational note (§6.4, `[ASSUMPTION]`-tagged) claiming Tier-2 default status that the actual gate (`matches_trigger_ruleset`, §5) does not implement. This isn't four scattered omissions — it's one systemic exclusion of a whole Key family whose entire purpose is "a scene just concluded," which is precisely the "after a battle / after a contest" seam this audit targets.

**`env.crisis` vs `env.peninsular_strain_shock`.** Trigger #8's condition text names the *specific* type `env.peninsular_strain_shock`, not `env.crisis`. These are separate registered types (`key_type_registry_v30.md` §6). A war's outbreak, a plague, a schism — the single highest-stakes peninsula Key family — has no in-the-moment cut scene; it only reaches the player via the annual Tier-3 chronicle (§4.4 item 1 explicitly harvests it), meaning up to a full season of latency between the crossing and the player learning why.

**Settlement drift is worse than a missing trigger — it's a missing Key.** Every other UNRENDERED family above at least has a registered Key type sitting in the substrate, reachable by Tier 1 and eligible for Tier 3. Settlement Order/L-PS/revolt/auto-capture have **no Key type at all**: `settlement_layer`'s module contract emits only `env.population_change`; the revolt gate (`g_ord0`) and undefended-capture gate (`g_def0`) carry no `then: emit` clause whatsoever, and the doc itself never uses the words "Key" or "emit." Nothing in the causal graph, no Memory entry, no chronicle candidacy — a total blackout at the substrate layer, not the articulation layer.

## Threadwork note (P-14)

A 2026-07-04 same-day NERS audit already adjudicated two thread findings I would otherwise re-report (`designs/audit/2026-07-04-ners-qualitative-audit/01_workings/refutations/refute_articulation-thread-silence.md`, `refute_articulation-zero-thread.md`): the articulation layer's zero literal thread/coherence/dissolution tokens is real but non-load-bearing because articulation is deliberately a universal `*` subscriber (ED-936) and `scene.thread_operation` does register (ED-935). Rendering Crisis (ED-681) *does* render — richly — via four authored beats in `threadwork_v30.md §3.7`, but that path runs entirely outside the Key/causal-graph/chronicle stack: it is the protagonist's own played scene, not an articulation-mediated render of someone else's consequence.

That is exactly where P-14 (consequence renders in every mode) is only half-satisfied: protagonist-present threadwork renders richly; **off-screen/NPC-witnessed threadwork consequence does not reliably render at any juncture in the table above.** A Dissolution witnessed by NPCs the player isn't with, or a Calamity radiating per `designs/world/calamity_radiation_v30.md` into a territory the player is absent from, has no Tier-2 trigger (no thread row in §3.1) and no guaranteed Tier-3 slot (only Knot/Belief are named paragraph types, §4.4 item 4) — it falls through to whatever generic Tier-1 salience the affected NPCs happen to accumulate, with no anti-starvation escape hatch, since `unarticulated_weight` (§3.3) only re-scores an *already-firing* trigger and can never cause a non-matching Key to fire one (a mechanism gap the 07-04 dossier already flags independently at `dossier_articulation.md` §3). The RS/MS naming collision (threadwork_v30 §5 "MS" vs `params/threadwork.md` "RS" for the same track, per grounding-03 §1) compounds this: it's unclear which track, if either, would even host a future dedicated thread Key — a prerequisite unresolved before any new trigger could be authored. Net: prior-audit verdict (REFUTED/P3/DELIBERATE for the "silence = defect" framing) stands; the residual — off-screen thread consequence has no rendering path — remains real, narrow, and UNDETERMINED-intent, and this audit's own settlement/env.crisis/standing_change findings suggest it is one instance of a broader pattern: **the trigger table was built sim-first around the 8–10 cases Stage 10 measured, not swept exhaustively against the full 44-type registry.**

## D6 stress-test: the case against P0 ‚Üí P1 ‚à• P3-lite ‚Üí P2 ‚Üí P4

**Strongest case for building Stage 4 (the four games) first**, argued as hard as I can make it:

1. The lane's own ratified workplan (`designs/workplans/valoria_master_workplan_v5.md:47,56`, row #15/#39 ‚Üí LA-19) and `finding_status.md`'s KT-1 both say Stage 4 is "ratified-as-next" ‚Äî this is not a rogue plan the audit is second-guessing, it's Jordan's own docket. An audit recommending a re-sequencing bears a burden the workplan doesn't currently carry.
2. P1 (the consequence spine) is not pure connective code ‚Äî `scene_dispatch.py:18-24`'s own header calls actor-derivation from aggregate World state "an undefined design decision" the module deliberately refuses to invent. That is real, open-ended design authorship with no existing answer, not a wiring task with a known shape. Estimating it as "Medium ‚Äî connective code" (the report's own words) may understate it; it could stall exactly like the four-games build would.
3. Three of the four Stage-4 games (Consensus/Negotiation/Inquiry) do not obviously depend on domain-echo keying (N-4) at all ‚Äî Consensus promotes existing `faction.py` vote code, and the interaction-shape work (venue/adjudicator/style mechanics) is orthogonal to whether Debate outcomes key by band or genre. So P0's blocking claim over Stage 4 is overstated for 3 of 4 games; only the echo-wiring step, not the game-authoring step, needs P0.
4. Content is the corpus's own repeatedly-flagged highest-leverage gap (KT-1, PR #77's own headline verdict for this subsystem) ‚Äî an audit deprioritizing it needs a strong reason, since "one game wearing eight venue skins" is arguably a worse player-facing defect than an invisible reachability gap no one has hit yet (no campaign has run long enough to need Stage-4 games either).

**Does this survive contact with the evidence?** No ‚Äî on balance it does not overturn the report's sequencing, for reasons the evidence itself supplies:

- Point 1 is answered by the report's own provenance check, which I independently reproduced: `HANDOFF_SC.md`'s "NEXT: Stage 3" is stale by exactly one stage (Gate C, dated 2026-07-02, already closed out Stage 3's armature/CR4/CR5 work per `GATE_C_packet_raw.json`) ‚Äî so "Stage 4 ratified-as-next" is the current true state, not a settled prior judgment the audit is overriding; it's simply the next unstarted stage on a docket that never once considered reachability. The workplan rows I read (`valoria_master_workplan_v5.md:21,47,56,71,85`) confirm this: they discuss content configuration exhaustively and never once mention scene_dispatch, domain_echo, or campaign reachability. The "ratified plan" the counter-argument leans on was never asked the question this audit asks.
- Point 3 is real but partial: even if 3 of 4 games don't need P0's echo decision, all four (including the promoted Consensus) still route through the same `scene_dispatch`/`run_contest` dead path N-1 describes ‚Äî so Stage 4 would ship three-to-four new game-shapes that a campaign still cannot reach, on top of the one that already can't be reached. Building content atop a confirmed-unreachable substrate doesn't convert "engine with excellent posture" into "game" ‚Äî it just adds more posture. This is the report's central point and it holds up: the two problems (reachability, content breadth) are not substitutes, and reachability is the more binding one because it gates whether *any* of the content, present or future, is experienceable in a campaign.
- Point 2 is a fair caveat on cost, not on ordering: even granting the derivation bridge is harder than "Medium," that argues for treating P1 as a real, possibly-multi-session design docket (which the report already frames as "genuine design work," not a quick fix) ‚Äî it does not argue for building Stage 4 first, since Stage 4 doesn't remove the need to eventually solve P1, and shipping four unreachable games first only compounds the same debt the report identifies.
- Point 4 conflates "highest content leverage" with "highest viability leverage." KT-1's leverage claim was made without anyone having checked reachability; once N-1/N-5 are on the table, "most valuable thing to build" and "most valuable thing to fix so anything can be experienced at all" are different questions, and the report is answering the second one, which it explicitly scopes as prior to the first (P2 still lands, just after P0/P1).

**Verdict on D6: the recommended P0 ‚Üí P1 ‚à• P3-lite ‚Üí P2 ‚Üí P4 sequencing survives.** The strongest available counter-argument (follow the ratified workplan, build content first) rests on a workplan that never considered the reachability question the audit surfaces, and even a partial version of the counter-argument (3-of-4 games don't need P0) doesn't escape the deeper problem that none of the four games would be campaign-reachable regardless of build order. I could not find an in-canon safeguard or ratified decision that treats the reachability gap as an intentionally-deferred, already-scoped-for-later item (unlike, say, ED-297's coalition dominance) ‚Äî `scene_dispatch.py`'s own comments flag the gaps as honest deferrals, not as a staged rollout plan with reachability scheduled for a later gate. The sequencing recommendation is upheld.

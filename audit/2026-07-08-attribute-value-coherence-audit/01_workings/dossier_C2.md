# Dossier C2 — Derived Values (agonist pass)

**Headline.** The derived-value layer is coherent at its newest, most-worked seam (personal combat: Health/Stamina/Wound-Interval/Max-Wounds agree byte-for-byte across `derived_stats_v30.md` §4.1, `module_contracts.yaml`, and `combatant.py`) but rots the further a quantity sits from that seam: `params/core.md`'s own master "Derived Scores" table — the exact prose layer CLAUDE.md §5 tells engine work to resolve numbers from — carries a **second-generation stale Health formula** (post-dates the ED-826 fix it should have inherited, now itself superseded by today's-adjacent ED-1021 D-A revision), is **missing Concentration entirely**, and the Mandate formula the board-game params layer (`factions_personal.md`, `bg/tracks.md`, `bg/core.md`, `bg/ed_resolutions.md`) still teaches is a **full ruling-generation behind** the settlement-level size-weighted aggregate that `module_contracts.yaml` and `descriptor_registry.yaml` already carry correctly. A brand-new (2026-07-08, same-day) ruling — ED-PC-0005 retiring the wound −1D pool cut — is already self-contradicted **within the same source file** (`combat_v30.md`) six lines apart. The census's own AMBIGUOUS Concentration row rests on a bad citation: `params/core.md:162` is the Thread Fatigue row, not a Charisma×3 Concentration row — there is no third Concentration formula, only a missing one. Garrison Strength/Local Economy/Public Order are formula-coherent everywhere they appear, but the home design section (§9) that originates them is explicitly flagged "PENDING... Not canonicalized" while the contract layer already gates on them as settled fact, with no ED ever closing that gap.

## Trace narrative by quantity

### Health
- Authoritative spec: `designs/scene/derived_stats_v30.md:55-90` (§4.1) — `WI = round(Endurance+4+0.4×Spirit)`, `MW = min(floor(Endurance/2)+1,3)`, `Health = round(WI×(MW+1)+0.25×Strength×Endurance)`, D-A/ED-1021 (2026-06-18) Spirit/Strength low-weight terms, felling at cumulative-damage ≥ Health.
- Sim: `designs/scene/combat_engine_v1/combatant.py:17-42` — `wound_interval()`, `max_wounds()`, `health_full()` implement the identical formula, with named constants (`SPI_WI_W=0.4`, `STR_HEALTH_W=0.25`, `WOUND_INTERVAL_BASE=4`) citing derived_stats_v30 §4.1/ED-1021 directly. **Matches.**
- Contract: `references/module_contracts.yaml:810,826-829` (personal_combat) — `derivations: output Health, formula "round(round(End+4+0.4*Spirit)*(min(floor(End/2)+1,3)+1)+0.25*Strength*Endurance) - cumulative_damage"`. **Matches** the design/sim pair exactly.
- Gap (census F-C-003, confirmed): the contract's `state:` list at :819-825 is `Health, cumulative_damage, Wounds, Stamina, Initiative, Poise` — **Endurance/Spirit/Strength never appear**, yet the Health `derivations:` block lists them as `inputs`. The character-attribute model those inputs come from is never cross-referenced anywhere in this contract. Key-binding-gap, confirmed accurate.
- **NEW, distinct from F-C-003**: `params/core.md:158` — the canonical prose "Derived Scores" table (the one CLAUDE.md §5 says to resolve numbers from) still reads `(End+6) × (MW+1)`, range `14–48` — this is the **ED-826-era fix** (2026-05-15, which replaced the even-older `Endurance×10` "Vitality" formula), never updated again after ED-1021 (2026-06-18) added the Spirit/Strength terms and shifted the real range to ~13–55 (per the derived_stats_v30 per-Endurance reference table, End 1→13, End 7→55). The row's own Notes column says "See `derived_stats_v30.md` §4.1 as authoritative" — an acknowledgment that this row is a **pointer, not the truth**, but it is formatted and placed exactly like every other live formula row, with no strikethrough or superseded marker. This is the *same defect class* ED-826 already fixed once, recurring one canon revision later, with no ED tracking the current gap.
- **NEW, live, same-day**: `designs/scene/combat_v30.md:301` states the ED-PC-0005 ruling correctly (fractional Ob only, no pool cut — "supersedes the −1D Combat Pool cut of PP-232... Jordan ruling 2026-07-08, ED-PC-0005"), but six lines later `combat_v30.md:307` — the "Hybrid" paragraph — still asserts *"Wound −1D Pool penalty carries into TTRPG mass battle Command checks (per `derived_stats_v30.md` §4.1 universal rule; PP-716 supersedes prior +1 Ob framing)"*, citing the now-doubly-superseded PP-716 −1D rule as if live. ### Mandate (faction scale)
This is the deepest finding of the pass — the census's "two derivations" framing undersells it; there are (at minimum) **three generations** of Mandate formula live in the corpus simultaneously, and the params/ prose layer never received the last rewiring:
1. **Pre-split (unsplit Mandate)** — oldest, pre-PP-686.
2. **PP-686 v2 (2026-05-01), flat faction-level split**: `Mandate = round(0.5×Legitimacy + 0.5×Popular_Support)`, with Legitimacy/Popular_Support as **faction-level** 1–7 stats. Still live verbatim at: `params/factions_personal.md:8,14` ("~~Mandate~~ (derived)... retained for backward compat"), `params/bg/tracks.md:1`, `params/bg/core.md:160` (a "Faction Starting Stats" table that still lists Legitimacy/Popular_Support as faction columns), `params/bg/ed_resolutions.md:30,99` ("derived-Mandate order = round((L+PS)/2)").
3. **LPS-1→LPS-2e (Jordan structural ruling, 2026-05-30, `designs/territory/settlement_layer_v30.md:147-179` §1.8)** — L/PS **demoted to per-SETTLEMENT** values; Mandate is a **size-weighted saturating aggregate**: `q_s = 0.5L_s+0.5PS_s`, `W_s = base(Type)+Prosperity_s+FacilityTier_s`, `T = Σ W_s·(q_s/7)`, `Mandate = clamp(round(7T/(T+6)),0,7)`, K=6. Explicitly stated to **supersede the faction-level L/PS canonized in PP-686 v2** ("those put L/PS at the wrong level"). This is what `references/module_contracts.yaml:87,598-600` and `references/descriptor_registry.yaml:70` ("Mandate is a size-weighted derived aggregate of settlement L/PS... NOT a base attribute") correctly carry.

The rewiring sweep that propagated LPS-2e is documented at `references/canonical_sources.yaml:205-208,346,383` and `designs/audit/2026-05-28-resolution-diagnostic/ledger_candidates_consolidated.json:596-599` — and it names its touched surfaces explicitly: `faction_behavior_v30 §2/§3.4/§3.5/§3.6/§4`, `faction_canon_v30 §3.4/§5.1-5.3`, `faction_state_authoring §8`, `stats_1_7_scale.md` (6-stat header, confirmed current at `params/factions/stats_1_7_scale.md:3`), `derived_stats_v30` footnote. **`params/factions_personal.md` and the `params/bg/*.md` board-game files are never named** in any of the three LPS-1/1b/2/2b/2c/2d/2e sweep notes, and on inspection they still teach generation-2 (PP-686 v2) as current, with zero pointer to the LPS-2e supersession. This is precisely the params/ layer CLAUDE.md §5 tells a number-resolving session to trust once it has found the subsystem head — and it is a full ruling-generation stale.
- **Adjacent KNOWN-TRACKED item, distinct surface**: ED-FA-0004 (2026-07-07) already flags that `sim/` oracle code (`game_state.py`, `faction_action.py`, `crown_initiative.py`, `accounting.py`, `parliamentary_vote.py`, `territory/registry.py`) implements the **pre-LPS-1** model and has been banner-flagged pending Stratum-B implementation. That ED covers the executable sim/ surface only — it does not mention or cover `params/factions_personal.md` or `params/bg/*.md`, which is prose the sim doesn't even read but a human/Godot-importer resolving "what is Mandate" from the designated params layer would.
- The `Legitimacy` name itself is separately overloaded: derived_stats_v30 §8/§14.1 defines a **faction-level derived meter** `Legitimacy(derived) = Mandate × 20` (0–140) — a *different* quantity from `set.legitimacy` (settlement-scale, 0–7, registered in `descriptor_registry.yaml:78`). `settlement_layer_v30.md:177` explicitly reconciles this ("the ×20 meter is a per-system derived buffer... not bound to 0–100") but no registry key distinguishes the two same-named "Legitimacy" referents at two different scales.

- The `g_dv0` gate (`module_contracts.yaml:575-578`) doesn't specify which of the three derived values triggers it or whether all three must independently hit 0 — gate-logic ambiguity noted, not fully adjudicated (census-flagged, left as-is per lane scoping — this belongs to the SE/gate-family docket, not ruled here).

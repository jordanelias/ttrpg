# PC — Personal / Scene Combat — Pessimist Subtractive NERS Dossier

**Lane:** PC · **Sources read:** `designs/scene/combat_v30.md` §§1–12 (esp. §4 ACTIONS, §5 weapon/damage, §8 group combat, §10 Thread-in-combat); `designs/scene/combat_engine_v1/{README,wrapper,state_graph,combat_throughlines_v1,systems}.py|.md` (CANONICAL, ED-900/904/ED-1029); `sim/personal/combat.py` (DEPRECATED, superseded 2026-06-23); `references/module_contracts.yaml` `personal_combat` entry; `canon/editorial_ledger.jsonl` ED-834/835/838/1041/1061; `references/throughlines_meta.md` §0 N, §1 Ω, §5 Q, §7 Failure Lexicon; audit charter `00_charter.md`.

## Reconciling the two objects

**Object A — the prose menu** (`combat_v30.md` §4): 12 named, declared, simultaneously-resolved actions (Strike, Feint, Establish Distance, Take a Breath, Full Guard, Disarm, Retrieve, Tie Up, Escape, Rescue, Dodge, Stunt), each with its own resolution-order slot and its own PP-citation trail. Plus a 13th, `Leap`, a Thread-practitioner action appearing in the same resolution-order table (priority 5) and detailed in §10.1 — extended into scope per the charter's "dossiers extend."

**Object B — the code resolver** (`combat_engine_v1/`): a continuous state graph (`state_graph.py` STATES) with **no discrete player-action concept** — `wrapper.engagement` resolves measure, commit-depth, mode (parry/dodge/wind), bind, and contact every beat as physics, not as a declared menu choice. `module_contracts.yaml`'s `personal_combat.modules` list names the canonical 11-module decomposition the engine hosts (`combat.strike` PORTED, `combat.wound` PORTED, `combat.feint`/`bind`/`approach`/`displace`/`grapple`/`halfsword`/`initiative`/`poise` PENDING, `combat.armour_defeat` FOLDED) — these are **engine internals**, not player verbs, but they are the best evidence available for whether a prose verb models something real or something the continuum already produces for free.

Two design-intrinsic facts from Object B directly falsify parts of Object A, independent of any build-state question:

1. **Feint was deliberately dissolved, not merely unbuilt.** `combat_throughlines_v1.md:39-41`: *"A feint is not a separate maneuver. It is the minimal-commitment/full-recovery pole of [the commitment=recovery] axis... This is why feinting was dissolved into the attack (WS-5) — there was never a separate thing to model."* This is an N-grounded design ruling on the *concept*, not a porting-priority note.
2. **The grapple family was deliberately unified, not merely unbuilt.** `state_graph.py` INJECTION_POINTS `contact.axis` describes the Contact node's outcome space as *"a flat branching menu (disarm/throw/pin/control/foot_pin/escape)"*, and `wrapper.py`'s `CONTACT AXIS` comments explicitly frame this as replacing **"three parallel grab checks"** with **"a unified `opening_created` flag... consumed once."**

Both rulings are cited as **design precedent**, not as "it isn't ported yet" — per the Cardinal Rule, build state is not itself evidence; the *reasoning* the canonical designers used to reach these rulings is.

A third piece of evidence complicates rather than simplifies: **ED-834/ED-835/ED-838** (2026-05-15 sim campaign) empirically found the "Strike/Feint/Full Guard action triangle" load-bearing for balance — an underdog (Agility 3, pool 11D) using Feint flips a 96.7%-loss matchup to a 65%-win one against a raw-Agility-advantaged opponent (Agility 6, pool 17D). Taken alone this looks like a hard KEEP for Feint on Ω-d grounds (non-dominance corrective). But the *specific* dominance vector that finding corrects — Agility inflating pool size — was severed nine days later by **R1** (2026-05-29 ratification, `combat_v30.md` §1: "Agility-INDEPENDENT resolution pool... supersedes PP-247's pool clause... closing the cross-attribute pool-dominance defect C-04"). The balance problem Feint was empirically shown to solve no longer exists in the form measured. What survives is the *tactical value* of a low-commit option against a raw-stat favorite — which Object B already supplies continuously (commit-depth 2 = the feint point on the axis) without a bespoke action. This dossier treats ED-834 as real evidence, weighs it, and still reaches DISTILL below — not a dismissal, a reconciliation.

A fourth structural tell: the prose menu's **own resolution-order table already clusters** actions that this dossier independently flags for merge, *before* any code comparison: priority **3** groups `{Disarm, Tie Up, Retrieve}`; priority **4** groups `{Establish Distance, Escape}`. The document's own taxonomy anticipated the redundancy.

## Per-action analysis

### 1. Strike — KEEP
The base action: allocate pool split, roll, apply damage (Impact×Coupling×Quality, D1-ratified). Models the irreducible core of the subject matter (melee/ranged exchange) with no substitute. Ω-b (permanent transformation via Wounds/death), Ω-c (NPCs strike autonomously in mass-battle handoff and background scenes), Ω-a (scene.combat_resolved/felled emit up to faction layer and NPC memory per `module_contracts.yaml`), Ω-d (every strike costs pool allocation traded against defence — no free action). Q-elegant: the rule restates in one sentence. Passes N/Ω/Q under attack. No finding.

### 2. Feint — DISTILL (fold into Strike's commit-depth declaration)
**Design intent:** let a fighter spend pool to threaten, provoking a defensive reaction and degrading the opponent's pool next round, independent of landing a real blow.
**As-if-built:** if built exactly as specified, Feint requires its own versus-roll, its own non-stacking rule (PP-293), its own floor (PP-294), its own expiry-on-incapacitation clause (PP-293), and its own resolution-order slot — six sub-clauses of bespoke machinery to produce an effect (a low-commitment threat that costs the opponent tempo) that Object B's canonical resolver already produces continuously from `commit_depth` (Beta-distributed 2–5) with **zero additional rules**: commit≈2 already *is* the feint, for free, every beat.
**Criterion:** N — Abstractable (a more general rule, the continuous commit axis, covers it with no lost decision); reinforced by Duplicate Coverage against the Strike pool-split itself (Feint is just a low-N-dice Strike declaration with an extra versus-roll bolted on).
**Countervailing evidence weighed:** ED-834/835/838 showed a Feint-shaped mechanic is empirically load-bearing against pool dominance — but that dominance vector (Agility→pool) no longer exists post-R1 (2026-05-29). The *value* (an underdog's tactical lever) is preserved by folding Feint into Strike as a declared low-commit allocation (e.g., "allocate ≤3 dice to Offence" already IS what Feint requires as its floor) rather than cut outright.
**Verdict:** DISTILL — remove Feint as a separately-declared, separately-resolved action; express the same tactical choice as the low-N end of Strike's own pool-split, exactly as the canonical resolver already treats it.
**Retires:** `module_contracts.yaml` `personal_combat.modules['combat.feint']` (resolver: d_sigma, status: PENDING) is itself a stale manifest row given the WS-5 dissolution already executed in `wrapper.py` — this verdict recommends striking that PENDING port task rather than ever porting a standalone Feint module, and drops PP-212/238/277/291/293/294's bespoke resolution text from any future `combat_v30`→`_v40` re-authoring pass.

### 3 & 9. Establish Distance + Escape — MERGE into one "Reposition/Disengage" action
**Design intent (both):** win an Agility contest against the opponent to control the spatial relationship — Establish Distance to a preferred in-scene range band, Escape to leave the engagement outright.
**As-if-built:** both resolve identically — Agility contest vs. opponent, no other mechanic differs — and the document already schedules them at the *same* resolution-order priority (4). The corpus already has exactly the tool needed to unify them without losing the outcome distinction: the Degree Table (Partial/Success/Overwhelming) used elsewhere for Feint/Disarm. A single "Reposition" declaration where Partial shifts one range band and Overwhelming breaks contact entirely reproduces both current effects with one contested roll instead of two.
**Criterion:** N — Duplicate Coverage (two actions modeling the identical dynamic — controlling distance — under two names). Reinforced diagonally by Object B: the continuous `Approach`/reopening machinery (`measure_gap`, `reopen_prob`, `reach_threat`) already treats "closing" and "creating separation" as one continuous spectrum, not two discrete verbs, further evidencing that the prose's two-verb split is an artifact of table authorship, not of the underlying dynamic.
**Verdict:** MERGE — one action, Degree Table determines magnitude (partial reposition vs. full disengage).
**Retires:** shrinks the `combat_v30`→`_v40` re-authoring task by one full rule-block (Escape's own dedicated paragraph + its own resolution-order row); aligns with the `combat.approach`/`combat.displace` PENDING modules' remit as the eventual code-side unification target.

### 4. Take a Breath — REFINE (Rest-state risk; needs a forcing function)
**Design intent:** let a fighter recover Stamina at the cost of forgoing offence, gated on not being in melee contact.
**As-if-built:** as a bounded, contested resource-management choice this models a real dynamic (fighters do recover breath between bouts) and Ω-d is respected (full opportunity cost, gated by range). The failure risk is structural, not a build-state complaint: nothing in §2/§4 prevents two ranged-postured combatants from mutually declaring Take a Breath indefinitely with no forcing function pulling them back into engagement — the Failure Lexicon's own "Rest state" fails Ω-c/Μ-α ("pressure is continuous... forces engagement over idling") by name, and this is exactly that shape if both sides decline to close.
**Criterion:** Ω-c / Μ-α (Rest state) per the canonical Failure Lexicon.
**Verdict:** REFINE — keep the recovery mechanic (its cost/benefit is sound), but add an explicit forcing function against mutual, indefinite standoff (e.g., an escalating scene-pressure clock, a cap on consecutive declarations, or tying eligibility to the same Approach/measure continuum Object B already tracks so "not in melee contact" isn't a stable, cost-free steady state).
**Retires:** none named (ordinary finding, not a scope cut).

### 5 & 11. Full Guard + Dodge — DISTILL/MERGE into the base per-round pool-split
**Design intent:** an "all dice to Defence, no attack" declaration — general (Full Guard) and Ranged-attack-specific (Dodge).
**As-if-built:** §3 Initiative already requires *every* combatant to declare an Offence/Defence pool split each round regardless of which named action they take. "All dice to Defence, 0 to Offence" is therefore already achievable as the floor value of the base split — Full Guard adds a name and a "cannot attack" clause to an outcome the base mechanic already permits; Dodge is the same floor value again, narrowed to ranged attacks (a distinction the existing Ranged-defence-at-Close-zone TN-8 rule and armour-DR-applies-normally clause already carry without a separate action).
**Criterion:** N — Abstractable/Duplicate Coverage (both are the 0-Offence endpoint of the mechanic every round already requires); reinforced Q-smooth concern (special-cased: two named rows exist to express one value of one existing variable).
**Countervailing evidence weighed:** ED-834 names "Full Guard" as part of the empirically load-bearing action triangle — taken seriously, this verdict does **not** remove the *choice* (0-Offence/full-Defence remains fully available every round); it removes the redundant standalone naming and the duplicate Dodge variant, which cost nothing to fold since the base split already carries the option.
**Verdict:** DISTILL (fold both into the base per-round pool-split as its documented floor case, differentiated by attack-type via existing TN/DR rules, not by two extra named actions).
**Retires:** shrinks §4's ACTIONS table by 2 rows and the deprecated `sim/personal/combat.py`'s `ACTION_RESOLUTION_ORDER` dict by the same 2 entries in any successor Tier implementation.

### 6, 7 & 8. Disarm + Retrieve + Tie Up — MERGE into one "Grapple/Control" action
**Design intent:** close-range contested maneuvers that disable a weapon (Disarm), recover a dropped one (Retrieve), or bind/immobilize (Tie Up).
**As-if-built:** Disarm and Tie Up share an identical template — Close range, single Offence roll, binary threshold, one named debuff applied — differing only in which debuff fires; Retrieve exists solely as Disarm's dependent follow-up clause, not an independent decision. Object B's canonical Contact Axis (`CT.grab_outcome`) already resolves this entire family as **one** contested roll branching to `{disarm, throw, pin, foot_pin, control, escape}` — explicitly built to replace "three parallel grab checks," per `wrapper.py`'s own commentary. The prose's own resolution-order table already co-locates all three at priority 3, independently corroborating the same grouping.
**Criterion:** N — Duplicate Coverage (three templates modeling one dynamic — close-quarters control — under three names, each with its own PP citation trail: PP-213 for Tie Up alone).
**Verdict:** MERGE — one "Grapple/Control" action, Close range, single contested roll, outcome table (disarm / bind-control / mutual-penalty) mirroring the code's already-ratified branching design; Retrieve folds in as the dependent recovery clause of that outcome, not a standalone row.
**Retires:** collapses 3 rule-blocks (Disarm, Tie Up, Retrieve) and their PP-213/PP-2xx citation trails into 1 for any `_v40` re-authoring pass; directly aligns the discrete menu with the already-planned `combat.grapple` PENDING module rather than requiring that module to separately reconcile three legacy prose templates when it eventually ports.

### 10. Rescue — REFINE (concept KEEP; rule text fails Q-elegant)
**Design intent:** let a bystander intercept an attack aimed at an outnumbered ally — a real small-unit combat dynamic, and the corpus's own designated answer to Multi-Engagement (3v2, 4v3) group fights.
**As-if-built:** the *concept* passes N cleanly (protecting a comrade under fire is load-bearing, not edge-case, and it is the sanctioned tactical answer to the Fibonacci group-bonus dynamic in §8) and Ω-b/Ω-d (real transformation risk to the rescuer; real cost — dice committed away from own defence). But the **rule as written** carries eligibility (PP-290, outnumbered-at-declaration only), a commit-and-contest formula (PP-292), payoff/exemption clauses (PP-292/295/406/407), a chain-block rule (no re-Rescuing a Rescuer), and a special incapacitation-before-resolution carve-out — roughly eight independently-citied sub-clauses. A designer unfamiliar with Rescue cannot restate its core rule after one reading, which is the literal Q-elegant test ("core rule restatable after one reading... external dependencies enumerated; 'except when X' flagged and justified" — this rule is almost entirely "except when X" clauses).
**Criterion:** Q-elegant fail (special-cased) per Failure Lexicon.
**Verdict:** REFINE — keep Rescue as a concept; consolidate the eight sub-clauses into a single contested-roll template (eligibility as one flat precondition, payoff/exemption as one resolved-outcome branch) rather than a sequence of patches accreted PP-by-PP.
**Retires:** none named (ordinary finding, not a scope cut).

### 12. Stunt — REFINE (live GM-arbitration contradicts the no-GM mandate)
**Design intent:** reward using terrain/position for a combat edge — a real, historically-grounded tactical dynamic (cover, high ground, improvised leverage).
**As-if-built:** the concept passes N (environmental advantage is load-bearing, not fantasy-imposed). The **mechanism** as written cannot be realized by any engine as specified: *"+N dice to Offence... Game Master sets N, max 5"* requires a live human adjudicator's judgment call, in a project whose foundational premise (`CLAUDE.md` line 5) is **"There is no GM — the engine resolves everything."** This is not a build-state gap (an as-yet-unautomated GM call would be a resolution-plan wiring item); it is a design-intrinsic defect — the rule's core variable (N) has no engine-computable definition at all in its current form, and this exact class of defect has an established, ratified fix pattern already in the corpus: **ED-1061** flagged "Guilds boost — GM picks" as *"a genuine defect under the no-GM mandate"* and replaced it with a deterministic, context-derived rule (adjudicator's rhetorical-appeal weighting) with no randomness and an explicit tie-break. The vocabulary-debt sweeps ED-765/768 (PP-678) already tracked and remediated 17+ similar "Game Master" references elsewhere in the corpus — Stunt appears to be a residual instance the sweep missed (it's in §4's ACTIONS table, not the passages the sweep's grep-based approach targeted). PP-720's *"Game Master arbitration if multiple equidistant"* (Stress-FF secondary-target tiebreak) is the identical defect shape one section over.
**Criterion:** Q-smooth (external dependency — a GM — not merely unenumerated but structurally undefined for an engine); cross-references the no-GM foundational premise directly.
**Verdict:** REFINE — redesign N as a deterministic, context-derived formula (e.g., a function of the zone/terrain tags an encounter already carries, or the Stunt roll's own degree-of-success feeding a capped table), mirroring the ED-1061 precedent exactly; do not cut the concept, only the GM-dependent mechanism.
**Retires:** none named directly, but this finding should be filed into the same standing "Game Master → engine" remediation track as ED-765/768/1061 rather than treated as a novel one-off.

### 13. Leap (Thread-practitioner action, §10.1) — KEEP
**Design intent:** let a Thread-sensitive practitioner perform a substrate-level operation mid-combat, at a steep, explicit tactical cost.
**As-if-built:** this is the strongest performer in the entire action set against the canonical criteria. N: models a real load-bearing dynamic specific to this game's ontology (combat as thread-configuration violence, Foundations A1) — not edge-case, not abstractable, because nothing else in the set touches the Thread substrate mid-fight. Ω-a: unambiguous, *specified* cross-scale consequence — feeds mass-battle Coherence (−1 per operation, §A.10), TS-observer perception tiers (§10.2), and Cross-System Fire consequences up to faction/witness/Conviction-Scar systems (§10.3, §13.3 Death Cascade) — the one action in the set that clearly satisfies Ω-a as written, not just in aggregate. Ω-d: explicit, steep, quantified cost — "commits all pool to Defence... ~60% hit probability... Real tactical cost" — a textbook non-dominant trade, the opposite of cost-hidden. Ω-c: autonomous Thread-perception events fire on *any* practitioner's combat wound/death regardless of the acting player, per the TS-tier table. Q-elegant: the core rule ("all pool to Defence during Leap, real risk") restates in one sentence; the derived consequences (§10.2/10.3) are enumerated, not hidden.
**Verdict:** KEEP — passes N + Ω(a/b/c/d) + Q under adversarial attack, gated correctly (practitioner-only, not a universal action, which is itself appropriate scoping rather than a flaw).

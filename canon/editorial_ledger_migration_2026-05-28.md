# Editorial Ledger — JSONL Migration (2026-05-28)

**What changed.** The editorial ledger was migrated from 24 fragmented YAML files (three parallel archive schemes, six unparseable) into a single canonical `canon/editorial_ledger.jsonl` — one JSON object per line. Rationale and trade-off analysis: per session decision (text/git-diffable record with DB-grade uniqueness via a validator, instead of SQLite which is non-diffable and breaks the commit-gate/citation model).

**Migration result (verified lossless).**
- 602 distinct source IDs recovered from all 24 files (including per-entry recovery of the six malformed archives) → **605 JSONL lines** (602 + ED-865/866/867).
- **0 IDs lost. 0 duplicate IDs. All 605 lines valid JSON.** (uniqueness — the core fix — achieved.)
- 21 entries were unparseable at block level and recovered by regex (id/description/status); flagged `_migration_partial: true` — **need manual re-verification.**
- 94 IDs had genuinely different descriptions across files (true conflicts, mostly stale variants in the misnamed `editorial_ledger_archive_1001_1200.yaml`, which actually holds ED-5xx/6xx entries with internal duplicates). For each, the richest/latest version is the primary line; alternates are recorded in `_migration_alt` and flagged `_migration_flag`. **These are preserved, not resolved — they are Jordan's call.**
- New audit findings filed: ED-865 (GD-2 miscite), ED-866 (manufactured 0-P1), ED-867 (upstream P≈0). Status: open / jordan_decision: pending.

**Status of old files.** The 24 YAML/archive files are LEFT IN PLACE this commit (non-destructive). They remain the files the hooks/bootstrap currently read. Removing them requires Phase 2.

**Phase 2 (tooling cutover — NOT done here).**
1. Point ledger-reading hooks/bootstrap (`task_gate('editorial')`, compliance, fetch, `append_to_register`, `assert_unique_ids`) at `editorial_ledger.jsonl`; parse JSONL.
2. Add a validator hook: round-trip parse + global ID uniqueness + required-fields (the DB-grade integrity guarantee in text).
3. Remove/deprecate the 24 old YAML/archive files once hooks are cut over.

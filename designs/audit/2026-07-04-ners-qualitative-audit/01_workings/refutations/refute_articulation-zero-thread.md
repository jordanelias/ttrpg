## What I verified

### Part (a) — TRUE, but narrower than framed
Grep of `designs/articulation/articulation_layer_v30.md` for thread|coherence|rendering
crisis|dissolution|calamity|gap|leap|practitioner returns only ONE hit (L123, an unrelated
"closes this gap" idiom about belief-revision routing). So no thread-*specific* Tier-2 trigger
exists. The §3.1 ruleset (10 triggers) is scar/coup/succession/mission/betrayal/knot±/
peninsular-shock/cascade-cluster/belief-revised — no Gap/Rendering-Crisis/Dissolution row.
Confirmed and matches grounding 03 §3 + dossier §2.

BUT §5 wires articulation as a **universal Key subscriber** (`consumes: {type:"*"}`,
module_contracts L768; file §5 "subscribes to all Key emissions"). Thread Keys therefore DO
flow through: §4.4 Tier-3 chronicle paragraph type 4 renders "Knot formations and ruptures,
Belief revisions" — Knots are thread-bonded relationships (knots_v30/ED-912). Tier-1
Concern/Memory (§2.1–2.2, §5) index every Key. So thread content is NOT wholly unrendered.
What is absent is a *dedicated Tier-2 cut-scene trigger* — much narrower than "zero rendering
path."

### Part (b) — REFUTED by direct evidence
The load-bearing rhetorical claim is that ED-681's beats "have no rendering path." This is
false. `threadwork_v30.md §3.7 "Rendering Crisis — Narrative Structure (ED-681)"` (read L700–760)
IS the rendering path: four fully authored beats (Withdrawal / Knot Anchoring / Place Anchoring /
The Choice) each with scene structure AND mechanical rolls (Spirit checks TN7 Ob1/2, a resolution
roll). The section header literally reads "For videogame implementation, the mechanical checklist
above unfolds as four designed beats." These are **protagonist-facing played scenes** rendered by
the scene/threadwork engine directly.

The articulation Tier-2 layer is, by its own §3 definition, "a short (5–15 second) engine-rendered
moment showing a **non-protagonist** event." Rendering Crisis is the protagonist's "existential
season" (§3.7) — a foreground arc the player *plays*, not a background happening that needs
surfacing. Routing it through the articulation cut-scene sink would be a category error. So the
absence of an articulation hook for Rendering Crisis is category-appropriate, not a lost rendering
path. The finding conflates "articulation has no thread trigger" with "thread narrative is
unrendered" — the second does not follow from the first.

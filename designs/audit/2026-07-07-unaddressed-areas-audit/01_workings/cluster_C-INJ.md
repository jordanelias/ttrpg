## miraculous_event: declared vs exists vs reaches-play

**Declared (module_contracts.yaml:727-742).** `doc: designs/scene/miraculous_event_v30.md` (an inline comment at line 729 notes a prior verification pass found "doc:null was stale — the doc exists" but left the CANONICAL-vs-PROVISIONAL header contradiction unresolved). `scales: [personal, settlement, peninsula]`. `resolver: state_reader [ASSUMPTION] gated emitter: fires on high-Ob Mending near T15`. `consumes: []`. `emits: [{type: meta.miraculous_event, terminal: false}]`. `state: []`, `transitions: []`, `loops: []`. `gap_notes` (line 739) still read "system-vs-event-source classification [OPEN — Jordan]; home doc [GAP]" — the "home doc [GAP]" clause directly contradicts the doc-exists note two lines above in the same entry.

**The canonical source doc** (`designs/scene/miraculous_event_v30.md`) is a fully worked mechanism, far richer than the contract's single emitted Key suggests:
- Trigger: Mending at Ob ≥ 6 succeeds within Proximity ≤ 2 of T15 Askeheim (§20.2), radius/effects scaled by Ob (6/7/8+).
- Effects declared in prose: SA (Southernmost Awareness) +1 per present faction (§20.3), a −1 Proximity-Rating modifier for RS-band lookup (§20.4), +1 Settlement Accord at next Accounting (§20.5), a per-character Conviction/Certainty roll for witnesses present in the affected territory (§25), and faction-specific response pathways/Decision Events for Church, Hafenmark/Baralta, Varfell, Restoration (§22-24) touching CI, RDT/TD, PT, Mandate, Stability.
- None of SA, the RS modifier, Settlement Accord, or the Conviction-roll trigger appear as `state:` entries or additional `emits:` in the contract — the contract's footprint is one Key type, the doc's is roughly eight distinct state edges across five tracks.

**Registry schema** (`key_type_registry_v30.md:859-876`): `meta.miraculous_event` required fields `event_type, center_actor, witnessed_by`; optional `peninsula_visibility`; `emitting_systems: [miraculous_event]`; `consuming_systems: [faction_layer, npc_behavior, articulation]`. No provenance field (no `causes`/`origin_keys`/`cause_keys`) is declared for this type — see census below.

**Consumers wired in module_contracts.yaml**: `faction_state` (line 70) and `npc_behavior` (line 119) both list `{type: "meta.miraculous_event", from: [miraculous_event]}` in `consumes:`; `articulation_layer` reads it only via its universal `*` wildcard (line 768). No other module consumes it.

**What exists in code:** `sim/world/miraculous_event.py` is a 22-line stub. Its single function `trigger_miraculous_event(event_type, world)` unconditionally raises `NotImplementedError` (line 21). `canon/mechanics_index.yaml:828-833` confirms `test_status: not_implemented`. No trigger-evaluation logic (the Ob≥6/T15/Proximity≤2 condition from §20.2) exists anywhere in `sim/`.

**Traced path from injection to player-perceivable play (claimed vs actual):** Even stipulating the stub were implemented and correctly emitted `meta.miraculous_event`, tracing forward through the corpus's own wiring: the Key's only structural consumers are `faction_state`/`npc_behavior` (accounting-side state updates) plus articulation's blanket read. Nothing in `sim/autoload/scene_slate.py`, `sim/cross_scale/scene_dispatch.py`, or `sim/cross_scale/zoom_in_out.py` references `meta.miraculous_event`, `miraculous`, or any Southernmost/T15/SA concept. The design doc's own §25 mechanic ("When a named character witnesses Miraculous Event effects [...] present in affected territory during the season Miraculous Event fires") presupposes the player is *in a scene* when it fires — but no Zoom-In trigger, mandatory or world-state (§4.3.2/§4.3.3), names Miraculous Event, and `scene_slate`'s own contract entry (`module_contracts.yaml:342-357`) declares `consumes: []` — it has no Key-driven entry point at all, only a raw `queue_scene(scene_type, context, priority)` API fed by hand at call sites. **Verdict: even a fully-built resolver has no mechanized path to place a witnessed miracle in front of the player.** The chain terminates at faction/NPC bookkeeping and a generic articulation read; it never reaches the Scene Slate.

## Injection→scene seam verdicts

Per §4.3.3 (`scale_transitions_v30.md:142-152`), the five World-State Zoom-In trigger rows are: Clock Band Transition, NPC Conviction Crisis, Treaty Proposed or Broken, Territory Control Change, Warden Emergency. **Neither injector has a row here or in §4.3.2's eight mandatory triggers** (`scale_transitions_v30.md` §4.3.2, mirrored in code at `sim/cross_scale/zoom_in_out.py:149-174`). Code-side confirmation that §4.3.3 has no adjudication path at all: `sim/cross_scale/zoom_in_out.py:44` declares `WORLD_STATE_TRIGGER_PRIORITY = 1` but the file defines only `check_mandatory_triggers()` (§4.3.2, lines 141-174) — **there is no `check_world_state_triggers()` function**; the constant is dangling/unused. `sim/cross_scale/scene_dispatch.py:46-69` (`evaluate_triggers`) evaluates exactly one of the eight §4.3.2 rows ("Stability Crisis") against real world state and defers the rest as "not evaluable against the current aggregate world schema" (line 32); §4.3.3 is not referenced anywhere in that module. `sim/autoload/scene_slate.py` is a bare priority queue (`queue_scene`/`next_scene`/`clear`) with `consumes: []` in its own contract (`module_contracts.yaml:347`) — nothing wires a Key of any type into it; callers must invoke `queue_scene()` directly, and no caller anywhere in `sim/` passes a miraculous-event or crisis/disaster scene type. `_resolve_slot()` in `scene_dispatch.py:83-114` only has live resolvers for `scene_type in {"combat", "contest"}`; any other `scene_type` (including a hypothetical `"miraculous"`) falls through to `"resolver for scene_type=... not live (stub or unmapped)"`.

**Verdict: for both injectors, the seam from injection to player-perceivable play does not exist at any layer** — not in the trigger tables (prose), not in the trigger-checking code, not in the scene-queue wiring, and not in the scene resolver dispatch. This is a strict superset/sharpening of KNOWN ep-29, not a contradiction of it.

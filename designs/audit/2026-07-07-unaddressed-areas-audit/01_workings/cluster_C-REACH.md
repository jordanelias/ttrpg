## Honest gaps

- **Dynamic/reflective dispatch not ruled out.** All reachability claims rest on static `grep` for import/call sites plus targeted empirical probes (up to 20 seeds × 50 seasons for the highest-stakes claims, single-seed for others). `faction_action.py`'s dispatch is a plain `if/elif` on `faction.name`, and `scene_dispatch`'s is a plain `if/elif` on `scene_type`, so I'm confident there's no `getattr`/string-registry indirection hiding an additional call site for the modules verdicted ISLAND — but I did not grep for `getattr(`, `importlib`, or `globals()[`-style indirection repo-wide to formally rule it out.
- **`tests/` were used only to confirm "zero callers," not read in full.** For modules where I reported "test-only" or "zero callers including tests," I grepped call-site patterns across `tests/` rather than opening every test file; a test that constructs a call via a fixture/factory indirection could in principle be missed.
- **Empirical probes are seed-bounded.** The `Turmoil`-never-written and `world.npcs`-never-populated claims are backed by both full-repo static grep (strong) and empirical runs (corroborating, not exhaustive) — I did not run the full `sim/tests/` regression suite or a large batch (100+ seeds) to hunt for a rare code path that populates these.
- **`designs/scene/combat_engine_v1/`'s internal correctness was not assessed** — only its *external* reachability from `sim/`. Its own test suite (`tests/valoria/test_combat_*.py`) and workbench were not exercised; I cannot speak to whether it works in isolation, only that nothing in the production loop calls it.
- **Faction-territory-loss edge case not explored.** `_faction_actions_callback` skips any faction with `not faction.territories` — I did not trace what happens once a faction is reduced to zero territories (does it permanently stop acting for the rest of the campaign, compounding degeneracy?) — flagged as related to C-REACH-7 but out of this pass's depth budget.
- **`§4.3.3 world-state triggers`** are referenced by a priority constant in `zoom_in_out.py` but I found no trigger table or evaluator for them anywhere — I did not exhaustively search `designs/` for whether they're specified but simply un-ported (vs. never designed), since that's more C-STUB's remit than C-REACH's.
- **No narrative/rendering layer audit performed beyond existence-check.** Confirmed no chronicle/narrative_engine consumer exists in `sim/` for any subsystem's output; a deeper audit of the narrative-engine v2 seam is explicitly out of this cluster's scope per the charter (gap class 4, "needs its own strong-critic session").

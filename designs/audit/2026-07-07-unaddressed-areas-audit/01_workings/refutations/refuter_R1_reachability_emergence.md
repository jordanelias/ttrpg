# Refuter R1 — reachability/emergence claims

_Archived verbatim from the agent's final message (2026-07-07)._

**Contamination disclosure:** a broad repo-wide grep for "87%" (done to find the provenance of CLAUDE.md's win-share figure) incidentally surfaced snippets from `finding_status.md` and other 2026-07-04/07-05 audit docs that state conclusions matching claims 3 and 7 almost verbatim (e.g. "44/44/12/0 at seed 0", "insurgency dead-by-construction"). This happened *after* I had already independently derived the matching numbers via direct simulation, so my conclusions are not downstream of that exposure, but flagging it for transparency per the relay-independence protocol.

---

**Claim 1 — Turmoil/PS gate vacuous.** VERDICT: **CONFIRMED**. `sim/autoload/victory.py:73` reads `world.clocks.get('Turmoil', 0.0)`; `sim/autoload/game_state.py:215` initializes `'Turmoil': 0.0`. Exhaustive grep of all `.clocks[` writes in `sim/` finds writers only for `'MS'` (ms_track.py:69,90), `'CI'` (ci_track.py:165), and `MASS_SEIZURE_FLAG_CLOCK` (mass_seizure.py:213) — never `'Turmoil'`. `sim/peninsular/accounting.py`'s 5-step season cascade (CI, MS decay, insurgency triggers/promotion, NPE) has no Turmoil step. Empirically verified: `Turmoil` stayed `0.0` across all 50 seasons of a live campaign run. **Intent gate: NOT-INTENDED.** `designs/provincial/peninsular_strain_v30.md` §4 fully specifies Turmoil's mechanics (battle +1/season, faction-elimination +2, revolt +1, decay −1/peaceful season) as CANONICAL; sibling clocks CI/MS each got dedicated implemented track modules (`ci_track.py`, `ms_track.py`) wired into `accounting.py`, but no `turmoil_track.py` exists and nothing wires it in. No ED/HANDOFF entry flags this as a deliberate deferral — it reads as a silent implementation gap, not a design choice.

**Claim 2 — Hafenmark 0-territory trap.** VERDICT: **QUALIFIED**. The mechanism is real and confirmed: `sim/mc_v18.py:73` (`if not faction.territories: continue`) permanently skips a faction once it hits 0 territories; I found no counter-mechanic anywhere in `sim/` — `_try_conquest` (faction_action.py) needs the faction's own territories to compute adjacency targets, and the one module that could restore territory to a starved faction, `parliamentary_transfer.py`, is never called at all (see claim 6). Conquest itself (faction_action.py:178-186) has no last-territory floor, so a faction *can* be driven to 0. Hafenmark starts Mil=3.0 vs. others' 4.0, exactly at the conquest-eligibility floor (`faction.Mil < 3.0` fails) — confirmed via `game_state.py:40` — but Mil never decreases anywhere in `sim/` (only increases via Muster), so this fact is true but not actually load-bearing. However, "0% win-share" is not strictly true: independent `run_batch` runs at n=100 for both base_seed=42 and base_seed=0 gave Hafenmark **1%** (1/100), not 0%. Tracing one such win (seed 71) showed Hafenmark's territory count never actually hit 0 that game (min=1) — i.e., it won by avoiding the trap, not escaping it. So: the one-way trap is real, but Hafenmark's win-share is near-zero/degenerate rather than a mathematical, hard-guaranteed 0%. **Intent gate: UNDETERMINED.** Two conflicting threads: (a) `designs/audit/2026-05-28-resolution-diagnostic/sim_collapse_recovery_results.md` explicitly rules — citing Tainter — that "a polity with zero territorial base does not reconstitute" is "the correct, realistic gate," suggesting zero-territory-terminal is philosophically DELIBERATE; but (b) that canonical "Reconstitute" mechanic (Influence Ob4, 3 successes, requires ≥1 territory) is itself never implemented in `sim/` at all, and `faction_action.py:88,128` marks Varfell/Hafenmark's unique actions as explicitly "BLOCKED (Pass 2d/2e + contamination audit)" — an acknowledged incompleteness, not a balance ruling. No "faction elimination" detection exists either (ties to claim 1's missing Turmoil-elimination trigger).

**Claim 3 — Insurgency pipeline dead-by-construction.** VERDICT: **CONFIRMED**. `check_insurgency_triggers` (`sim/world/insurgency_pipeline.py`) requires ≥2 contiguous `owner is None` territories sustained 2 seasons. Exhaustive grep of all `.owner =` assignments in `sim/` finds only `t.owner = 'Church'` (mass_seizure.py:292) and `t.owner = faction.name` (faction_action.py:181) — never `None`. Only one territory (`T15`) starts as `owner=None` (`STARTING_OWNER`), and nothing ever creates a second. The canonical trigger mechanism itself — "Revolt" turning Accord-0 territories Uncontrolled (`peninsular_strain_v30.md` §4d) — has zero implementation in `sim/` (only docstring/comment mentions, no executable code). Empirically verified across 3 seeds × 50 seasons: `world.insurgencies` and `world.uncontrolled_streaks` stayed `{}` throughout; `T15.owner` stayed `None` untouched all game. **Intent gate: NOT-INTENDED.** `insurgency_pipeline.py`'s own docstring calls itself "PARTIAL IMPLEMENTATION" awaiting `designs/world/insurgency_pipeline_v30.md` (Pass 2i, "not yet authored") — an acknowledged-incomplete module, but nothing flags that its precondition is structurally unreachable given Revolt was never built.

**Claim 4 — generate_npc has zero call sites.** VERDICT: **CONFIRMED**. `generate_npc` is defined at `sim/world/npe.py:198`; repo-wide grep finds its only other occurrence is its own docstring entry-point listing (line 21) — literally zero callers anywhere, including tests. `world.npcs` defaults to `{}` (`game_state.py:162`) and `create_world` never seeds it. Empirically confirmed: after a 50-season campaign, `world.npcs == {}`. `simulate_npc_actions` (called every season from `accounting.py:79`) iterates `store.items()` over that permanently-empty dict — a confirmed no-op loop. **Intent gate: NOT-INTENDED.** The module's docstring frames `generate_npc` as a service entry point meant to be invoked by "higher-tier modules" (investigation/fieldwork scenes) — an unwired integration gap, not a decision to leave NPCs empty forever.

**Claim 5 — Nine World registries unreachable.** VERDICT: **CONFIRMED** (with a precision caveat). Direct empirical check after a full 50-season campaign: `practitioners`, `treaties`, `knots`, `convictions`, `beliefs`, `threadcut_beings`, `territory_infrastructure`, `npc_drift_state`, and `settlements` were **all** `{}` — none ever populated. Caveat on the "import graph" framing: static AST-based import tracing from `sim/mc_v18.py` *does* transitively reach the files that define `knots`/`treaty`/`infrastructure`/`coherence`/`threadcut`/`beliefs`/`conviction` — but only via `game_state.py`'s deserialize-path imports (never invoked by `mc_v18.run_campaign`, which only calls `create_world`/`serialize_world`) or late-imports inside function bodies that the live campaign loop never triggers (e.g. `contest_legacy_stub.py`'s belief/conviction hooks, gated behind `scene_dispatch.py`'s self-flagged "CONTEXT-DERIVATION BRIDGE GAP" which "resolves nothing"). So functionally the registries are unreachable, even though the raw files are technically import-linked. **Intent gate: NOT-INTENDED.** Every field comment in `game_state.py` names the owning module as if population is expected; `scene_dispatch.py` explicitly self-documents this as a named, tracked GAP ("context-derivation gap: ... resolves nothing"), not a deliberate end-state.

**Claim 6 — parliamentary_vote/transfer/stay have zero external callers.** VERDICT: **CONFIRMED — but KNOWN-TRACKED, not new.** Grep confirms zero callers of `propose_transfer`, `invoke_stay`, `resolve_stay_lift`, `run_parliamentary_vote` anywhere in `sim/` outside the three files' mutual imports. Critically, **`canon/editorial_ledger.jsonl` entry `ED-SC-0007`** (2026-07-05, RATIFIED post-merge by Jordan on PR #80) already states verbatim: *"grep confirms mc_v18 never reaches parliamentary_stay/transfer/vote"* — this is a pre-existing, ratified ledger finding, not a new discovery. Per the charter's non-duplication rule, this should be reported as KNOWN-TRACKED (ED-SC-0007), not NEW. **Intent gate: NOT-INTENDED.** ED-SC-0007 itself frames this as a "P1 consequence-spine work item (accepted)" — an acknowledged gap Jordan has already accepted needs wiring, explicitly not a deliberate choice.

**Claim 7 — ~87% is a base_seed=42/small-n artifact.** VERDICT: **CONFIRMED.** Traced the "~87%" figure to `designs/audit/2026-06-30-ecosystem-adversarial-review.md:75` ("`mc_v18.run_batch(8, seed=42)` yields Varfell 87.5%/Church 0%/Hafenmark 0%"). Independently reran `run_batch(8, base_seed=42)`: reproduced **exactly** — `{'Crown': 12.5, 'Church': 0.0, 'Hafenmark': 0.0, 'Varfell': 87.5}`. Reran `run_batch(16, base_seed=0)`: got `{'Crown': 43.8, 'Church': 12.5, 'Hafenmark': 0.0, 'Varfell': 43.8}` — matching the claimed "44/44/12/0" to within rounding. At larger n=100 for both seeds, neither approaches 87% (seed42: Varfell 56/Crown 36/Church 7/Hafenmark 1; seed0: Varfell 54/Crown 40/Church 5/Hafenmark 1) — confirming 87% is specifically a tiny-sample (n=8) artifact, not a robust characterization. **Intent gate: NOT-INTENDED.** CLAUDE.md's framing states this as a settled current-state fact and it has propagated unchallenged into ≥5 other audit docs without anyone re-running at larger n or alternate seed — a documentation/methodology gap rather than a deliberate characterization.

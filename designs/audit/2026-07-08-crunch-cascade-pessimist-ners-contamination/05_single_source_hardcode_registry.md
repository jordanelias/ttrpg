## SECTION 3 — STRUCTURAL PATTERNS (read this first for an architectural fix)

The registry's failures cluster into seven recurring structural causes. Fixing the *cause* retires whole columns of the registry at once.

**P-1. A generator/exporter that reads only part of its own domain.** The archetype (Worst Offender #1): `export_engine_params.py` draws its scope at `config.py` and never reads `core.py`, so the Godot seed omits the pool formula. The *mechanism* is a hardcoded module boundary (`import config`) that is narrower than the concept the artifact claims to represent ("the combat oracle"). Same shape latent in any future params importer scoped to one file. **Architectural fix:** generators must enumerate their domain by *concept coverage* (a manifest of every constant the artifact must contain), not by which module was convenient to import; the `--check` round-trip should fail if a canonical-tagged constant anywhere in the oracle package is absent from the export.

**P-2. Auto-extracted registries whose generator has gone dead, still read as truth.** `values_master.yaml` (extractor `extract_values.py` dead; 71 refs to a nonexistent file; quarantined but still cited). This is the CLAUDE.md §8 "tool exists ⇒ rule enforced" fallacy applied to *data*: the file looks structured and authoritative, but nothing regenerates it, so it is a fossil. **Architectural fix:** any generated data file must carry (and CI must verify) a live regeneration command; a file whose generator is dead is deleted, not quarantined-and-retained.

**P-3. Hand-authored term registries never re-derived from the code/docs they describe.** `glossary.md`, `mechanical_terms_index.md`, `alias_registry.yaml`, `censured_vocabulary.yaml`, `synonym_registry.yaml` were each authored once and drift-audited by eye. This produces the single largest drift surface in the registry: glossary is stale on Health, Stamina, Intelligibility, Composure, Focus, Command, Standing, roster-count, the Overwhelming BG formula, "Memory"-vs-Recall; it is *missing* Bonds, Face, Stasis, Adjudicator Armature, Morale, Resonant Style; and its collision table omits disp/Disposition, Standing (3 senses), TD (Theological Dissatisfaction), CE (surveillance), RS (Resonant Style), Momentum (Institutional), Stamina (scale). `mechanical_terms_index.md` copies glossary's stale readings verbatim (shared-ancestor duplication). `censured_vocabulary.yaml`'s residual counts (CP=0, RS=1) are wrong because it searched full phrases, not abbreviations. **Architectural fix:** the term registries should be *generated* from the design/params/code heads (the inverse of today's `build_lexicon`, which flows registries→lexicon); or, minimally, a CI check that every `## Status:`/ED-ratification touching a term forces a glossary diff in the same commit (CLAUDE.md already states this convention — it is unenforced).

**P-4. Two (or more) registries each asserting authority, with no deferral edge between them.** descriptor_registry(9) / glossary(7) / params/core.md(10) / alias_registry(7) for the attribute roster; three Standing ranges; `canonical_sources.yaml` self-contradicting on Piety Track; `canonical_registry.md` (RDT struck) vs `params/bg/tracks.md` (RDT active). No file "defers to" another, so `build_lexicon` ingests all of them and faithfully propagates the conflict. **Architectural fix:** exactly one head per concept, and every other file carries a *pointer* (the `names_index.yaml`→`descriptor_registry.yaml` MIRROR relationship is the model) — enforced, not conventional.

**P-5. Old files never deleted after supersession, still read or imported.** `sim/personal/combat.py` (deprecated, live-imported by `scene_dispatch.py`); `combat_design_v1.md` (orphaned, no banner, carries Strength=Size and CP=Combat-Power); `combat_v30.md` (Status line says "working" while banner says superseded); `sim/provincial/massbattle.py` (frozen v22); `threadwork_superseded.md` (feeds `values_master`); `derived_stats_v1` cited but nonexistent; `valoria_ttrpg_complete.md` cited but nonexistent; `params/combat.md` cited 71× but nonexistent. A newcomer greps, finds the stale file, and trusts it. **Architectural fix:** supersession must either delete the file or add a loud banner *and* break/redirect every import and citation to it (dangling citations to nonexistent files are a CI-checkable class).

**P-6. A markdown params file and a code config each restate the same constant with no build-time sync — and the canonical head contradicts *itself*.** Cross-file: `params/core.md` vs `combat_engine_v1` constants (Health/Stamina/Wound-Ob hand-kept in sync); `params/scale_transitions.md` vs `scale_transitions_v30.md` (Domain Echo cap/target drifted). Intra-file, the more insidious variant: one canonical head disagrees with itself — `derived_stats_v30.md` (Health §4.1 vs §3/§13), `threadwork_v30.md` (Focus §2.3 vs §2.4; Certainty §646 vs core.md), `victory_v30.md` (Rupture), `params/contest.md` (Orientation), `params/scale_transitions.md` (Echo cap line 34 vs 42), `articulation_layer_v30.md` (CANONICAL vs PROVISIONAL status lines), `params/bg/ed_resolutions.md` (Overwhelming :83 vs :157). **Architectural fix:** the recommended typed engine-params YAML (CLAUDE.md §5) with CI round-trip *is* the fix for the cross-file half — provided it obeys P-1 (covers the whole domain); the intra-file half needs a "one value stated once per doc" lint.

**P-7. Renames executed in one direction only; collision-detection tables that are themselves hand-authored-once and incomplete.** Presence→Charisma (live formulas yes, glossary/alias_registry no); Cognition→Acuity & Spirit→Will (registry yes, everything downstream no); Cohesion→Discipline (bodies yes, `params/mass_combat.md` headings no); RS→MS (design head yes, params/spec siblings no); Contact Rounds retired (names_index yes, params/core.md no). And the safety net — glossary Part Twelve — is itself a hand-authored-once table (same as P-3) that misses the live collisions listed there. **Architectural fix:** a rename is a single atomic operation with a completion check (grep the old token to zero across live files) before the ledger entry flips to resolved; collisions get *generated* from a token-frequency scan, not hand-maintained.

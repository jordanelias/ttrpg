## Reconciliation intro

FI's player-action homes span two documents that do not fully agree with each other, plus one sim stub file that mirrors the design's system boundaries without adding new verbs:

- **`designs/scene/fieldwork_v30.md` + co-file `fieldwork_investigation.md` §4** — the six canonical Fieldwork *Investigation* actions: **Examine, Interview, Research, Surveil, Thread-Read, Reconstruct** — each a single roll against a shared Depth/Ob/degree table (§1–§4.2), each producing evidence tagged by reliability (§4.3).
- **`designs/scene/investigation_systems_v30.md`** (Status: CANONICAL, approved 2026-04-17) — four systems layered on top: **NPC Population Engine (NPE)**, **Investigation Interface** (Scene-as-Graph, Case Board, Zoom-In triggers), **Dialogue Lattice** (gated player utterances), **Response Resolution Matrix** (five-filter NPC response chain). Of these, only the **Dialogue Lattice's utterance-selection** is itself a player verb; NPE (world generation), the Response Matrix (NPC-side filter chain), and the Sincerity Gate (auto-triggered check) are load-bearing *machinery*, not actions the player takes, so they are folded into the Dialogue Lattice's analysis rather than given their own rows. Traversal of the Scene-as-Graph is explicitly declared by the source doc to add "no new action vocabulary" — node interaction *is* one of the six Fieldwork actions, just given spatial expression — so it is not a separate row either.
- **`threadwork_v30.md` §175-178** cross-references Thread-Read as fieldwork's sole Thread-substrate operation (fires co-movement, P-01) — read jointly with fieldwork's §4.5 for that action's verdict.
- **`sim/personal/investigation.py`** and **`sim/personal/fieldwork.py`** are both Pass-2l armature stubs (`NotImplementedError` on every entry point) that mirror the design's system names 1:1 — there is no code-side divergence in *vocabulary*, only in *depth* (the stub exposes 3 generic engine hooks, not the 6 named actions as an explicit enum). `module_contracts.yaml` has **no** `investigation`, `dialogue_lattice`, or `fieldwork` module entry at all (only the unrelated `fieldwork_knots` module, which governs Knot lifecycle, not investigation actions). `CURRENT.md` carries no fieldwork/investigation row — open per `ED-IN-0016` (2026-07-05, RATIFIED-AS-ACCEPTED), which explicitly names **EP-8: "investigation_systems_v30-vs-fieldwork_v30 contradiction"** as an unsettled head-conflict. This is the FI-lane instance of the charter's "prose-menu vs code" divergence instruction: here it is **prose-vs-prose** (two co-equal design docs both claiming resolution authority over the same verb, Interview), and it is the single largest reconciliation problem in the lane.

**Prior audit material folded in:** the 2026-06-10 full-depth fieldwork diagnostic (`archives/audit/2026-06-10-investigation-exploration-diagnostic/`) already re-derived architecture findings INV1–INV9 and returned **N: PASS** for the engine as a whole ("no redundant roll... the deterministic pipeline owns disclosure/stance decisions") — a strong prior signal that the *shape* of the six-action system is sound. It flagged **E: FAIL (P1)** and **R: FAIL (P2)** purely on canonical-indeterminacy grounds (three docs give two different Thread-Read attributes, two different Disposition→Ob rules) — spec-consistency problems, not design-merit problems, and are noted below only where they bear on a specific action's Q-elegant read. `ED-913` (open, P1) already documents the Thread-Read attribute drift (Spirit is canonical per PP-619/PP-626; `fieldwork_investigation.md` and `params/fieldwork.md` still say Attunement). `ED-921` (open, P2) already documents the Interview/Lattice schedule-and-attribute mismatch. Neither prior finding proposed a *subtractive* disposition — that is this audit's addition.

## Per-action analysis

### 1. Examine
**Design intent:** Study physical evidence, architecture, documents, objects on-site (Cognition; §4.2). Produces **Verified**-tag evidence — usable as court evidence, Contest corroboration, Domain Action justification (§4.3).

- **N:** Passes. Forensic/physical-evidence examination is a genuine Renaissance-political dynamic (documents, seals, architecture as evidence of conspiracy or succession fraud) — not fantasy imposition, not an edge case.
- **Ω:** (a) cross-scale — Verified evidence feeds Findings that can trigger Domain Echo at faction-level scope (fieldwork_v30 §2.5) and is admissible in Contest/Tribunal, a genuine cross-scale payoff. (d) pays what it buys — Ob scales with Depth, Failure costs +2 Exposure, contestable via the opponent's Concealment Pool (§4.6). No dominance.
- **Q:** Elegant and smooth — one-sentence rule ("roll Cognition vs Depth-Ob, get Verified evidence"), and it shares its roll/degree/Exposure spine with Research/Interview/Surveil without special-casing (Q-smooth win, not a flaw — see Research below for the adversarial attack on this shared spine).
- **Attacked as duplicate of Research/Interview?** Yes — deliberately, given the pessimist mandate. All four mundane actions share an *identical* Ob table, degree table, and Exposure schedule (fieldwork_v30 §4.2); they differ only in (attribute, fictional source, reliability tag). Attack fails: the reliability tag is not flavor, it is a real downstream admissibility fork (§4.3 — Verified vs Testimonial vs Documentary have different weight in Contest/Tribunal), which is exactly the "meaningfully different player situation" the N-test (§0 Q3) demands. This models a genuine real dynamic (physical evidence outranks hearsay in an inquest) rather than duplicating one.
- **Verdict: KEEP.**

### 2. Research
**Design intent:** Consult archives, oral histories, institutional records, Einhir documents (Recall; §4.2). Produces **Documentary**-tag evidence (+2D Recall bonus when cited in Contest, §4.3).

- Same N/Ω/Q read as Examine — institutional record-keeping and archival research is a load-bearing Renaissance dynamic (chancellery records, notarial archives as instruments of political leverage), distinct fictional locus (off-site institution vs on-site object) and a distinct reliability tag with its own downstream mechanical payoff (Contest Recall bonus).
- Same adversarial attack (duplicate-coverage vs Examine/Interview) and same failure of that attack, for the same reason.
- **Verdict: KEEP.**

### 3. Surveil
**Design intent:** Observe a location, person, or faction operation over extended time (Cognition; §4.2). Flat **+2 Exposure** regardless of roll degree (the only mundane action with a fixed rather than degree-scaled cost) — Investigation Interface (System 2) additionally grants Surveil spatial expression as the "Observation" node type.

- **N:** Passes — surveillance/watching-and-waiting is a genuine tradecraft dynamic, named explicitly in the charter's player-action-home list.
- **Ω:** (a) cross-scale — Observational-tag evidence is usable as faction-level intelligence (§4.3). (d) pays what it buys, and pays *differently*: a guaranteed Exposure surcharge in exchange for information without confrontation, versus the other three mundane actions' degree-contingent Exposure. That is a real, distinct strategic trade-off (certain-but-costly vs variable-but-cheaper-on-success), not a copy of Examine/Research's shape.
- **Q:** Elegant (restatable in one line) and robust (a genuine third viable approach alongside "examine the evidence" and "ask the witness" — watch and wait).
- **Verdict: KEEP.** This is the cleanest possible rebuttal to the duplicate-coverage attack on the mundane-action cluster: its cost-shape, not just its flavor, is load-bearing.

### 4. Thread-Read (investigation variant)
**Design intent:** A perceptive Leap (Spirit ×2 + History + TPS, PP-619/PP-626) into Thread-level configurations for investigative purposes; TS ≥ 30 required; fires co-movement (P-01) — "the only fieldwork action that produces co-movement" (fieldwork_investigation.md §4.5). Cross-referenced at threadwork_v30.md §175-178.

- **N:** Passes trivially and necessarily — Thread substrate ontology (Μ-γ) is Valoria's constitutive frame, not an optional flourish; an investigation system with no Thread-level evidence route would be the actual N-fail (abstracting away the setting's central ontology).
- **Ω:** The strongest clause-pass in the entire lane. (a) cross-scale — co-movement's three-dimensional auto-effects (temporal/epistemic/actualized, threadwork_redesign_v25 §3.2) ripple into Calamity Drift, Certainty shifts, and a d6 consequence table — genuinely traceable-but-not-fully-anticipable, textbook Ω-a. (b) personal transformation — Certainty modifiers fire on use. (c) autonomous — co-movement's auto-effects are partly independent of player intent. (d) pays what it buys with real, asymmetric, *interesting* costs: +1 Exposure, Coherence cost at Relational+ scale, and — the standout design touch — Thread-verified evidence is **institutionally inadmissible in Church Tribunal** even though it is factually real (§4.3), and delivers only half-value to non-sensitive investigators (P-08 epistemological barrier). That is a genuine irreducible trade-off (truth vs institutional usability), exactly what Ω-d asks for.
- **Q:** Elegant — restatable in one breath ("TS≥30 characters can Leap-read Thread evidence directly, at the cost of co-movement, Exposure, and institutional inadmissibility"). Robust — dramatic legibility is sharp (what's at risk is both the co-movement fallout and whether the evidence will even be usable where it matters).
- **Reconciliation note (not a verdict driver):** the primary attribute is a live, open contradiction — fieldwork_v30 §2.1/§4.2/§4.5 and threadwork_v30 §175-178 both say Spirit (canonical per PP-619/PP-626); `fieldwork_investigation.md` §4.2/§4.5 and `params/fieldwork.md` still say Attunement (ED-913, open, P1, propagation-only — "Resolution is evidence-decided, not a Jordan choice"). This is a spec-propagation bug, not a design-merit question, and does not move the verdict.
- **Verdict: KEEP** — the action that best exemplifies the lane's Ω clauses as designed.

### 5. Reconstruct
**Design intent:** Synthesize accumulated evidence into a conclusion (Recall; §4.1/§4.2). Ob = threshold − progress (min 1) — cost falls as investment rises. Produces a **Finding** at the reliability of its weakest constituent evidence. Failure produces a **plausible but wrong conclusion the player acts on without being told** (§4.1). The Investigation Interface's Case Board Thread Layer (ED-680) layers a TS-gated +1 insight bonus (max once) for linking rendered and substrate evidence of the same event.

- **N:** Passes — the synthesis moment (turning scattered intelligence into an actionable political conclusion) is the load-bearing climax of any real intelligence/investigative process, not an edge case.
- **Ω:** (a) cross-scale — a Complex/Structural Finding is explicitly usable as Casus Belli, Heresy evidence, or faction intelligence (Investigation Interface §Case Board), a strong cross-scale payoff. (b) personal transformation — the Failure state is the standout: the player is not told they are wrong, and *acts* on a false conclusion until events contradict it. That is a genuine personal stake with narrative teeth, not a null result. (d) pays what it buys — the falling-Ob-with-investment curve rewards prior work without eliminating the real risk of Failure.
- **Q:** Elegant, robust, and smooth — "assemble the evidence, but if you got it wrong you won't know until it bites you" is one-read legible and the dramatic stakes (what happens if you act on a wrong conclusion) are named in the spec itself. The Thread Layer insight bonus is a small, well-scoped addition, not scope creep — it composes cleanly rather than special-casing.
- **Verdict: KEEP** — arguably the single strongest, most game-defining action in the lane (explicitly modeled on *The Case of the Golden Idol*'s synthesis beat); survives the adversarial attack cleanly.

### 6. Interview (fieldwork_v30 §4.2, single-roll object)
**Design intent (Object A):** Question a witness/informant/subject; Attunement; identical Ob/degree/Exposure spine to Examine/Research/Surveil; produces **Testimonial**-tag evidence (−1 reliability if witness hostile).

**Reconciled against Object B — Dialogue Lattice utterance-selection (investigation_systems_v30 System 3):** The *same source corpus* declares this object superseded: investigation_systems_v30's own Cross-System Integration Table states "Interview action now routes through Dialogue Lattice + Response Matrix **instead of** single Charisma roll" (itself also mis-citing the attribute — Interview is Attunement, not Charisma, per fieldwork_v30 §2.1 — a second, compounding drift already caught by `ED-921`, open, P2). The design's own stated intent is *supersession*, not coexistence — yet both specs remain live and contradictory over a month later: `ED-921` documents the Lattice's flat +1-per-disclosure yield diverging from Interview's graded +0/+1/+2/+3 progress table, and the total silent disappearance of Interview's failure state and +2 Exposure cost once routed through the Lattice. `ED-IN-0016`'s EP-8 independently flags "investigation_systems_v30-vs-fieldwork_v30 contradiction" as an open head-conflict, unresolved because fieldwork/investigation has **no `CURRENT.md` row** to settle which document governs.

- **N — Duplicate coverage:** two live specs claim to resolve the identical player verb (probing an NPC conversationally) with incompatible mechanical shapes (rolled-graded-degree vs gated-flat-yield), and the corpus's own design intent says only one should survive. This is the textbook Failure Lexicon "Duplicate coverage" trigger, not merely an unfortunate accident of phrasing.
- **Ω:** judged in isolation, the bare single-roll Interview *does* clear a floor (Testimonial evidence, contestable via Concealment, admissible with Contest use) — it is not dominance or flavor-only. But that floor is **exactly what Object B (Dialogue Lattice) already covers and was explicitly authored to exceed** ("mechanically thin... one roll, no decision tree" is investigation_systems_v30's own stated reason for building the Lattice). Retaining both does not add a distinct Ω clause; it adds an unreconciled fork.
- **Verdict: MERGE / PRUNE** the bare-roll Interview spec into the Dialogue Lattice, once the schedule/attribute reconciliation is done — not a judgment that conversational investigation is weak (it is the opposite: the richer object should win), but that keeping *two* canonical specs for one verb fails N on duplicate-coverage grounds under the pessimist default. `intent_gate: NOT-INTENDED` — the design text is explicit that supersession, not duplication, was intended; the two-object state is an unexecuted propagation gap, not a deliberate two-tier design.

### 7. Dialogue Lattice utterance-selection (investigation_systems_v30 System 3, incl. Response Matrix + Sincerity Gate as supporting machinery)
**Design intent:** The player selects among gated conversational utterances (7 gate types: Attribute, Evidence, Belief, Certainty, History, Disposition, Thread Sensitivity), producing one of 6 outcome types (Evidence yield, Disposition shift, Conviction engagement, Information reveal, Escalation trigger, Arc trigger). NPC response is filtered through a five-filter chain (Information → Conviction → Disposition → Compromise → Ethical Framework) reading the NPC's Genome (System 1/NPE) and Ontological Ledger. Sincerity Gate auto-fires on [INSTRUMENTAL]-tagged utterances.

- **N:** Passes — "exploratory dialogue distinct from adversarial resolution" is a genuine gap in a two-mode system (thin Interview roll / full adversarial Contest) and models a real dynamic: probing, building rapport, and testing positions before committing to confrontation is exactly how Renaissance-era political actors gathered intelligence and built alliances.
- **Ω:** Strong pass on multiple clauses. (a) cross-scale — utterance outcomes route into Evidence Track, Case Board, and (via Conviction engagement) the Response Matrix, which can produce Conviction Wounds and fire arc vectors — a rich, traceable-but-emergent consequence chain. (b) personal transformation — the **Certainty Gate is the standout mechanic in the entire FI lane for this clause**: it is explicitly "the ontological status gate" — a character who has seen the Thread literally cannot honestly speak pure Church orthodoxy, and vice versa. Dialogue options are gated by *who the character has become*, not merely what they know or roll. This is as close to a pure Ω-b mechanic as the corpus has. (d) pays what it buys — the Sincerity Gate imposes a real, automatic cost on instrumental utterances; Belief-gated utterances bypass it but require an active Belief (itself a scarce, opportunity-costed resource).
- **Q — the honest attack:** Q-elegant is genuinely strained. The worked "Full Chain Example" (investigation_systems_v30.md, Response Matrix §) takes five sequential filter evaluations to resolve *one* utterance, each filter itself branching on NPC Certainty/Conviction-wound-count/Disposition-range/Compromise-Profile-category/Ethical-Framework-alignment. "Core rule restatable after one reading" fails at the level of the *composed system*, even though each individual gate/filter is simple in isolation. In a "no-GM, engine resolves everything" videogame (per CLAUDE.md's framing), this composed depth must run silently and be compactly legible to the player — the design specifies the internal chain in detail but does not specify a compact result presentation (only "Visible-locked" hints for gates, nothing for filter-chain outcomes). This is a genuine Q-elegant fail on the composed system, not on any single gate.
- **Verdict: KEEP, with REFINE flagged on Q-elegant.** The design intent (character-state-gated dialogue, Ω-b at its strongest) is worth keeping and is not itself dominant, redundant, or flavor-only — it should survive adversarial attack on N and Ω. But the five-filter NPC-response chain should be simplified or given a mandated compact player-facing legibility layer (e.g., a single "why this NPC responded this way" readout) before it is built, rather than after. This REFINE does not retire a resolution-plan task (it is additive simplification guidance, not a scope cut) — reported as an ordinary finding per the charter's own allowance (§4).

Judged only as context for whether Dialogue Lattice utterances have a rich enough NPC-side substrate to justify their Ω-a/b claims; they do (Genome axes are exactly what the five-filter chain reads).

- **Scene-as-Graph traversal / Case Board / Thread Layer toggle:** the design explicitly states node interaction is "one Fieldwork action... no new action vocabulary" — traversal and the Case Board are presentation/economy layers over the six actions already analyzed, not new verbs requiring separate rows.

## Lane summary

The FI action set is unusually strong on Ω: Thread-Read and the Dialogue Lattice's Certainty Gate are two of the clearest Ω-a/Ω-b exemplars anywhere in the corpus, and Reconstruct's fail-forward false-conclusion mechanic is a model of Q-robust dramatic legibility. Examine, Research, and Surveil survive a genuine duplicate-coverage attack because their shared mechanical spine is differentiated by a real, load-bearing reliability-tag consequence rather than mere flavor. The one clean subtractive finding is structural, not conceptual: **Interview exists as two live, contradictory specs** (a graded single roll in fieldwork_v30, a flat-yield gated Lattice in investigation_systems_v30) where the corpus's own text says one should have already superseded the other — this is judged a duplicate-coverage MERGE, not a comment on conversational investigation's worth. The Dialogue Lattice's five-filter response chain earns a Q-elegant REFINE (composed complexity outstripping one-read legibility) without threatening its underlying KEEP. No action in this lane clears the bar for an outright CUT — the lane's failures are all reconciliation and legibility debt, not design-intrinsic uselessness.

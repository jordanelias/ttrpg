# Perimeter target-point / normal model — machine-vision validation

`perimeter_render.png` renders each engine formation (Line / Arrowhead / Column / GappedLine) with the
perimeter target-point model computed from its ACTUAL footprint (convex hull of the cells), overlaid:
- **yellow** = perimeter faces
- **big red** = major target points (face midpoints)
- **small red** = minor target points (corners); **orange** = sharp vertex (interior angle ≤60°, the
  pointed-formation exception)
- **blue** = outward normals (perpendicular at faces, corner-bisector at vertices) = the required approach angle

Generated by `perimeter_render.py` (`python3 perimeter_render.py` → HTML → headless-Chromium screenshot).

Machine-vision verified against Jordan's hand-drawing (2026-07-23): the **Line** panel matches the drawing
exactly (4 face-mid majors + 4 corner minors + perpendicular/diagonal normals). Findings: (1) sharp-tip
detection is correct — the Arrowhead's acute BASE corners flag orange while its ~82° apex (for this aspect)
does not; (2) the convex hull bridges the GappedLine's internal lane — gapped formations need concave /
internal-edge perimeter handling, not a convex hull. Spec: `audit/2026-07-22-mass-battle-stress-test/
perimeter_targeting_geometry_v1.md`.

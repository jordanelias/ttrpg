## SIM-ARC42-01: Debate Thread Co-Movement RS Recovery — Mode A Isolation

### Key mechanic: Memory win → RS +1 if Thread-sensitive participant present

**params_contest §Thread co-movement:**
"Thread co-movement: Memory win → RS +1 (retention invoked). Projection win → RS +1 if Thread-sensitive."

**Questions:**
1. "Win" = decisive outcome or any exchange win?
2. Per-contest or per-exchange?
3. Who must be Thread-sensitive — the winning orator, any participant, any audience member?

**Reading the text literally:** "Memory win → RS +1" and "Projection win → RS +1 if Thread-sensitive." The asymmetry suggests:
- Memory win: RS +1 regardless of TS presence (retention = invoking what was, which has Thread-level resonance intrinsically).
- Projection win: RS +1 only if a Thread-sensitive person is present (projection = arguing what could be, which requires a sensitive mind to anchor the new configuration).

**But Arc 42 states both require Thread-sensitive presence.** This is a reading error in the arc. Memory win appears to fire unconditionally.

**Finding: Arc 42 understates the RS recovery.** If Memory wins produce RS +1 without TS presence, then ANY parliamentary debate about Thread-relevant history that resolves with a Memory-genre win produces RS +1 — whether or not practitioners are in the room. This is more powerful than the arc describes, but also more aligned with the philosophical foundations (P-03: rendering is what minds do; debating history IS a form of rendering).

**Frequency question (ED-414):**
The text says "Memory win → RS +1" without specifying per-contest or per-exchange. A Grand Debate has 5 exchanges. If RS +1 fires per exchange won, a debate could produce RS +5 in a single scene.

**RS seasonal cap: ±10 net RS change per season (params_threadwork §RS Track).** RS +5 from a single debate is within cap but would be the single largest RS recovery event possible outside of WC ≥ 3 (+2/season). This seems mechanically significant enough to warrant per-contest rather than per-exchange application.

**Severity: P2 — ED-414 is confirmed as requiring resolution. Per-exchange RS +1 creates a powerful RS recovery exploit via repeated short debates. Per-contest RS +1 is balanced but underwhelming.**

**Proposed resolution:** RS +1 per contest (not per exchange). The co-movement fires on the debate's resolution, not on individual argument victories. This is consistent with P-01 (inseparability co-movement fires per operation, and a debate is one operation).

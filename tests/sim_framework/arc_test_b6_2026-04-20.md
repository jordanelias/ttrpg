# B6 — Cathedral Construction Acceleration (engine_v4.0)

**Date:** 2026-04-20
**Engine:** `tests/sim_framework/engine_v4.py` (new, per ED-724 — v4.0 foundational migration)
**Trigger:** ED-725 open item — "B6 should test the scale-up of dynamic SW: Church builds Cathedral in T8/T14 — does it push Piety Yield meaningfully?"

## Setup

Engine v4.0 introduces:
- `Settlement` dataclass (first-class entity)
- `Territory.sw` as derived property: `min(5, Σ religious_building_tier per settlement)`
- 34 settlements populated from `settlement_layer_v30 §1.6a` seed table
- New helpers: `sack_settlement_building(gs, sid, reason)`, `build_settlement_building(gs, sid, target, reason)`
- Backward-compat: legacy territory-level `sack_religious_building` / `build_religious_building` wrap the settlement-scale ops
- **Parity verified**: v3 vs v4 S1 CI delta identical (both +3, seed 42); all 16 territory SWs match geography seed table exactly.

## Findings

### B6-1 Baseline (10-season passive growth)
CI 28 → **65**, mean Δ **+3.70/season**. Same as B5-1 (v3/v4 parity).

### B6-2 Upgrade S-003 Valorsplatz Cathedral (Church → Cathedral) at S3
- Before: T1 SW=2 (S-003 as Church). After: T1 SW=3.
- CI @ S10: **61** vs baseline 65.
- Mean Δ (S4–S10): **+3.86/season** (+0.16 vs baseline, within noise)

**Finding: Upgrading infrastructure in Crown-controlled territory has zero immediate effect.** T1 is Crown-controlled with Church.M (5) = Crown.M (5) — Church is not Prominent at game start. The upgraded S-003 yields only if Crown M drops below 5 (e.g., via Royal Decree overreach, battle loss, parliamentary action). The +0.16 observed is noise from stochastic Assert/Suppress rolls, not a true Piety Yield gain.

**Strategic implication:** Building Cathedrals in non-Prominent territories is **preparatory investment** — it creates stored Piety Yield capacity that activates when Prominence flips. Church must either target its own territories (always Prominent) or pair infrastructure spend with Mandate-attack on the controlling faction.

### B6-3 Aggressive Expansion (Chapel in T1 + T5 + T6 at S2)
Built Chapels in S-001, S-009, S-011. T1 SW 2→3; T5 SW 2→3; T6 SW 1→2.

- CI @ S10: **61** (identical to B6-2; no improvement over baseline)
- Mean Δ (S3–S10): **+3.75/season** (baseline 3.70)

**Finding:** Even aggressive expansion across 3 territories provides only marginal (+0.05/season) CI gain when those territories are not Prominent. The whole SW investment is stored value pending Prominence flips.

### B6-4 SW Cap Verification
S-024 upgrade to Cathedral in T9 (already at SW 5): Σ(3 Cathedral + 2 Church + 3 Cathedral) = 8, capped at 5. **Cap enforcement working correctly.**

### B6-5 Rebuild After Sack
Sack S-023 at S5, rebuild to Cathedral at S8. T9 SW recovers to 5. Post-rebuild Δ = **+4.00/season** (slight bump vs baseline, within noise).

**Finding:** Rebuild via Ecclesiastical Appointment fully restores SW. No decay penalty or hysteresis.

---

## Disposition

**ED-722 dynamic SW validated at per-settlement scale.** The v4.0 model correctly implements ED-722 mechanics: construction / destruction events are localized to individual settlements, aggregated to territory SW via the property, and consumed by existing CI-generation code without further changes.

**Strategic asymmetry confirmed (amplifies B5-4 finding):**
- Church can only monetize Piety Yield in Prominent territories (Church.M > controlling.M)
- At game start, Church is Prominent only in its own T9 — so dynamic SW elsewhere is stored value
- Building Cathedrals preemptively is a **bet on future Prominence** (Mandate erosion of target controller)
- Conversely, Church's defense of T9 (Prominent by control) is its only immediately-productive SW asset

This is a meaningful design constraint. Church infrastructure investment has a two-step payoff: build (step 1) → destabilize controller's Mandate (step 2) → Prominence activates yield.

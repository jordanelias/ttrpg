## SIMULATION 30 — C-14: RISKBREAKER + THREAD OPERATIONS (Mode B)

### Archetype: Riskbreaker

Deniability Debt (DD) accumulates from covert operations. Thread operations are not covert operations by definition — they are Thread operations. But a covert Thread operation (concealed per §2.3) may interact with DD.

**Interaction chain:**
1. Riskbreaker performs a concealed Thread operation (sets concealment pre-Leap)
2. Concealment succeeds: operation undetected
3. DD: does a successful concealed Thread operation accumulate DD?

**Gap (SIM6-F-01, P2):** No explicit rule linking Thread operations to Deniability Debt. DD accumulates from covert actions — but "covert" is a deniability-system category. Thread operations with concealment are covert in effect but not formally categorised as "covert actions" in the Thread operation rules.

**Proposed resolution:** Concealed Thread operations accumulate DD only if they are performed in a deniability context (on behalf of a patron, against a target, as part of a contracted operation). A practitioner who quietly Weaves their own wound closed in a back alley accumulates no DD. A practitioner who covertly Dissolves a Lock on behalf of the Revolution against Crown infrastructure accumulates DD as a covert action. Context determines DD accumulation, not the Thread operation type.

**Riskbreaker at TS 50 (Stirring): feasibility check**
- Riskbreaker archetype + TS 50 = Approach Training required
- Standard Riskbreaker is not a practitioner archetype — they can have TS growth through confrontation, but it's incidental
- If a Riskbreaker has TS 50+ and Approach Training: they can perform Thread operations. Their DD accumulation from those operations follows the contextual rule above
- If detected despite concealment: CE accumulation for the Inquisitor; potential IP change if politically sensitive

**Finding: No mechanical conflict between Riskbreaker and Thread operations.** Gap is in explicit categorisation only. ✓ Flag SIM6-F-01 as P3 (text clarification, not play-blocking).

## SIMULATION 31 — C-15: KNIGHT TEMPLAR WITNESSING THREAD OPERATIONS (Mode C)

### Archetype: Knight Templar (Church military, Devout Constraint)

**Setup:** Knight Templar (Spirit 4, TS 0, Devout Constraint active) witnesses a Dissolution attempt that produces a Gap and Monstrous Incursion. Combat ensues.

**State before:**
```
Knight Templar — Coord 4, Spirit 4, End 3, Health 8/8, Wounds 0
  Devout Constraint: active
  TS: 0 | Certainty: 4
  Combat pool: Coord 4 + Weapon History 3 = 7D
```

**Event sequence:**

**1. Dissolution witnessed (Gap forms):**
TS 0: perceives nothing at Thread level. Observes physical consequences only (something tears open, reality distorts).

Certainty check: Spirit TN 7 Ob 1 + Devout +2D bonus = 6D. P(pass): ~92%. ✓

**2. Monstrous Incursion — combat:**
Knight Templar engages. Standard combat. No Thread interaction.

**Devout Constraint in combat with monstrous entity:**
The entity is Mode 1 or 2 (rendered, from the Gap). Knight Templar's Devout framework categorises it as demonic/divine test. No mechanical conflict — the entity is a combat target.

Per P-04: monstrosity ≠ moral. The entity is not evil by nature. But the Knight Templar's Belief frames it as evil. This is a Belief conflict with philosophical reality — not a mechanical issue, but a roleplay pressure point. ✓

**3. CE accumulation:**
Knight Templar witnesses the Thread operation that caused the Incursion. CE +1 (Thread op in scene). But: Knight Templar is a military operative, not an Inquisitor. Does CE accumulate for non-Inquisitor Church characters?

**Gap (SIM6-F-02, P2):** CE accumulation rules specify Inquisitors as the primary CE accumulation archetype. Non-Inquisitor Church characters (including Knight Templar) witnessing Thread operations — do they accumulate CE? If yes: every Church soldier becomes an investigation asset. If no: Knight Templar's Thread encounters have no institutional consequence beyond narrative.

**Proposed resolution:** CE accumulates for any Church-affiliated character who witnesses a Thread operation and reports it to Church authority (files a formal account). Unreported witnessing = no CE. Knight Templar in active combat may not file a formal account immediately. CE accumulation is gated on institutional reporting, not mere witnessing.

**4. Theological Dissonance check:**
The Gap is a rendering failure — Ein Sof visible through the tear. For a Devout character with TS 0, this is perceptually opaque (they see the physical effects only). No Discovery Event triggered (TS 0: no Thread confrontation possible). Devout Constraint unaffected. ✓

**Finding: Knight Templar is mechanically robust against Thread exposure.** Devout bonus protects Certainty; TS 0 prevents Discovery Events; combat capability handles Incursions. The archetype is well-insulated from Thread mechanics — which is correct design. ✓

## SIMULATION 32 — C-16: SEVERED PRACTITIONER (COHERENCE 1) EMERGENCY OPERATION (Mode C)

### Setup: Practitioner at Coherence 1 (Severed) must attempt an emergency Thread operation — the only one who can Mend a Standard Gap before it becomes Entrenched.

**State before:**
```
Practitioner — Spirit 4, Coherence 1 (Severed), Wounds 0
  Active penalties (Severed):
    −2D social, −2D Memory
    Dissociative episodes once per scene regardless of operations
    All Knots +2 strain per session
    Involuntary perceptual events
  Mending pool: Att 4 + Focus 3 + TPS 7 = 14D
  Ob: Standard Gap = 5 + RS Fragile +1 = 6

RS: 44 | Coherence: 1
```

**Leap eligibility at Coherence 1:**
Per §9.16 (P-27): Coherence 0 prevents all Thread operations. Coherence 1 = Severed, but above 0. The practitioner can still attempt the Leap.

**Leap roll:**
Pool: Att 4 + History 1 + TPS 7 = 12D, Ob 1 (TS 50+) + Severed... 

**Gap (SIM6-F-03, P2):** No Ob penalty listed for Severed (Coherence 1) in the Severed threshold row. Fragmented (+1 Ob) and Fractured (+implicit from Certainty reduction) have penalties. Severed is worse than Fractured but has no Thread operation Ob penalty listed beyond what Fractured already imposed.

Looking at the threshold table:
- Fragmented (4–3): +1 Ob Thread operations including Leap
- Fractured (2): Certainty maximum reduced, dissociative episodes on Thread operations
- Severed (1): dissociative episodes regardless of operations, Knot strain, involuntary perceptual events — **no Thread operation Ob penalty listed**

**Finding (SIM6-F-03, P1):** Severed (Coherence 1) has no Thread operation Ob penalty. A Severed practitioner operates at the same Ob as a Fragmented practitioner (+1 Ob from Fragmented, which persists at lower Coherence). This seems like a gap — Severed should be harder to operate from than Fragmented.

**Proposed:** Add to Severed row: "+2 Ob on all Thread operations including the Leap (rendering barely holds; suspension is constitutively dangerous at this state)." This replaces the Fragmented +1 Ob (not stacking — Severed is its own tier).

**Proceeding with Severed = +2 Ob (proposed) for this simulation:**

Leap: 12D, Ob 1 + 2 (Severed) = Ob 3. P(success): ~88%. Likely succeeds.

**Dissociative episode trigger:** Severed = dissociative episodes once per scene regardless of operations. Does this fire during the Leap? During contact?

**Gap (SIM6-F-04, P2):** Timing of Severed dissociative episodes relative to the Leap and contact is undefined. If the dissociative episode fires during the Leap round: does it disrupt the Leap (like a Wound)? During contact: does it drop contact (like a Wound disruption)?

**Proposed:** Dissociative episodes at Severed fire at the start of each scene, not during operations. The episode is a pre-scene rendering instability, not a mid-operation interrupt. The practitioner can operate within the scene after the episode resolves.

**Mending at Severed:**
Pool: 14D, Ob 6 + 2 (Severed) = Ob 8 (at Mending ceiling). P(≥8 at 14D): ~17%.

**Most likely: Failure.** Coherence −1 (cap) → 1→0 = **Rendering Crisis**.

**Rendering Crisis triggered mid-operation:**
At Coherence 0 (during contact): §9.16 — Coherence 0 prevents Thread operations. But the operation is already in progress. Does Rendering Crisis mid-contact immediately terminate contact?

**Gap (SIM6-F-05, P2):** No rule for Rendering Crisis occurring mid-contact. The practitioner Leaped at Coherence 1, performed an operation that cost −1 Coherence, and is now at 0 during contact. Proposed: Rendering Crisis mid-contact terminates contact immediately (rendering collapses — it cannot be suspended any further). The operation result is Failure regardless of the roll's actual outcome. The practitioner returns to rendering in Rendering Crisis state.

**Finding: A Severed practitioner attempting any Relational+ scale operation will almost certainly trigger Rendering Crisis.** The Coherence cost (−1) from the operation pushes them from 1 to 0. The only safe operations at Severed are Object/Personal scale with no residue (base cost 0). Any Relational+ operation is a Rendering Crisis trigger. This is correct — Severed practitioners are functionally retired from large-scale Thread work. ✓

## SIMULATION 33 — C-17: EHRENWALL — COUP TRIGGER + THREAD ENVIRONMENT (Mode C)

### NPC: Ehrenwall (Löwenritter, Coup trigger conditions, Martial Law)

**Unique mechanics:** Coup trigger conditions, Martial Law, Crown loyalty assessment.

**Thread relevance:** Ehrenwall is TS 0 (military commander). His interaction with Thread mechanics is environmental — the Thread environment affects his coup calculus.

**Coup trigger conditions (assumed from gap register/designs):** Ehrenwall moves toward coup when:
- Crown Mandate falls below threshold AND
- Military assessment favours Löwenritter AND  
- Legitimate succession path is blocked

**Thread environment interaction:**

RS at 35 (Fractured): Rendering failures occurring in Crown territories. Non-practitioners experiencing inconsistent memories, spatial unreliability. Crown's institutional legitimacy is undermined by substrate failure — people cannot reliably remember Crown proclamations, trust legal documents, or maintain consistent social contracts.

**Mechanical effect on coup conditions:**
- Crown Mandate: RS Fractured → faction Stability checks Ob 1 each season (§5.3 Critical band — wait, Fractured band doesn't have faction Stability checks; that's Critical 19–1). At RS 35 (Fractured): no automatic Stability checks. But the narrative rendering failures affect Crown's perceived legitimacy.
- At RS 19 (Critical): faction Stability checks Ob 1. Crown's Mandate could drop mechanically from RS degradation alone, independent of political action.

**Finding (SIM6-F-06, P2):** No explicit rule connecting RS threshold effects to faction Mandate or coup trigger conditions. RS 19 (Critical) triggers faction Stability checks — this mechanically pressures Mandate — but the coup trigger conditions don't reference RS state. A world at RS 15 (near-Rupture) should make coup conditions more likely (institutional rendering fails) but the mechanic doesn't connect these.

**Proposed:** Add to §5.3 Critical band: "Faction Stability checks failure produces Mandate −1 for that faction. At RS ≤ 10: coup-condition NPCs (flagged in their design files) gain +1 to their trigger check pool." This connects the world-degradation track to political instability mechanics.

**Martial Law interaction with Thread environment:**
Martial Law (assumed: suppresses movement, increases military authority). In a high-RS-degradation environment: Martial Law may be harder to enforce (rendering failures mean proclamations aren't reliably received, orders aren't consistently remembered). No mechanic for this exists.

**P3 (SIM6-F-07):** Martial Law enforcement difficulty under RS degradation is unaddressed. Low priority — GM can handle narratively.

## SIMULATION 34 — C-18: LENNETH — SCHOLARLY TS ACQUISITION (Mode C)

### NPC: Lenneth (Crown/scholarly, covert Revolution endowment, scholarly TS path)

**Unique mechanics:** Scholarly TS path (non-confrontation), covert networks, Revolution endowment, covert contact (Int Ob 3).

**Scholarly TS path test:**

Per §4.4 (from compiled Stage 2 characters): "A character with access to Einhir scholarship... may pursue TS growth through sustained intellectual engagement... Spirit check still required — intellectual understanding alone does not produce sensitivity."

Lenneth's scholarly path: studying Einhir texts + analysis of Thread-operation records. Each qualifying study season: Spirit check TN 7 Ob 1.

Lenneth Spirit (assumed 4): P(success per season) ≈ 80%. Expected seasons to TS 30: starting TS 0 → 30 = 6 qualifying events (+5 TS each) = 6 seasons at 80% success rate → expected ~7.5 seasons.

**Covert contact (Int Ob 3):**
Lenneth Intelligence (assumed 4): pool 4D, TN 7, Ob 3. P(success): ~40%. Marginal — covert contact is genuinely risky for Lenneth.

**Revolution endowment interaction:**
Lenneth endows the Revolution. This is a financial/Circles action — no Thread mechanics. IP may change (intelligence networks aware of the endowment). ✓

**Thread operation capability:**
Once Lenneth reaches TS 30 (after ~7 seasons): eligible for Approach Training via Einhir Texts path. Then: TS 30 practitioner. Object/Personal scale only initially.

**Finding: Lenneth's scholarly TS path is slow but reliable.** 7–8 seasons to TS 30. Non-confrontation path means no Discovery Events, no Devout Constraint interaction. Clean progression. ✓

**Gap (SIM6-F-08, P2):** No rule for whether Lenneth's scholarly TS path is available while simultaneously conducting covert operations. The scholarly path requires "sustained intellectual engagement" — presumably a full-season commitment. Can a character simultaneously pursue scholarly TS growth and perform covert actions (separate season activities)?

**Proposed:** Scholarly TS growth and covert operations are compatible within the same season (studying and operating are not mutually exclusive). The qualifying events are study sessions, not exclusive seasons. The Spirit check fires at season end regardless of what else the character did.

## SIMULATION 35 — C-19: RS CRITICAL — MULTI-FACTION ACCOUNTING (Mode C)

**State:**
```
RS: 18 | TC: 8 | IP: 25

  Varfell M3 R2 W4 S4 | Hafenmark M4 R3 W5 S3 | Niflhel M2 R1 W3 S2
Active Locks: 2 (both 3-season, −1 RS/season drift each)

```

**Accounting sequence:**

**Step 1: Thread operations this season**
- 3 TTRPG operations (1 success, 1 partial, 1 failure): RS −0, −1, −2 = −3
- Revolution Mend order (success): RS +1
- Varfell Weave order (partial): RS unchanged, Co-Movement card drawn

**Step 2: Lock drift**
- Lock 1 (3 seasons): −1
- Lock 2 (3 seasons): −1

**Step 3: Gaps persisting**
- Standard Gap: −4
- Micro-Gap: (Micro-Gaps not listed in RS degradation table — see SIM6-F-09 below)

**Gap (SIM6-F-09, P2):** Micro-Gaps are not listed in the RS degradation sources table (§5.2). The table shows "Gap persisting (per season): −4" — but is this all Gap types or only Standard+? Micro-Gaps are same-scene and should close without Mending in most cases. If a Micro-Gap persists to Accounting (practitioner failed to Mend it), does it cost RS −4?

**Proposed:** Micro-Gap persisting to Accounting: RS −1 (not −4; Micro-Gap is a small substrate breach). Standard Gap: −4. Larger Gaps: scale upward. Flag as **SIM6-F-09 (P2)**.

**Step 4: Winter drift**
RS −1.

**Step 5: Critical band effects**
- Spontaneous Gap check: 1d10, on 1–4 = Gap forms (doubled at Critical). P(Gap): 40%.
- All Thread operations +1 Ob worldwide (already applied above).
- Faction Stability checks Ob 1: each faction rolls.

**Faction Stability checks (Ob 1, at RS Critical):**

| Faction | Stability pool | P(pass) | On failure |
|---|---|---|---|
| Crown (S2) | 2D | ~59% | Mandate −1 |
| Church (S3) | 3D | ~73% | Mandate −1 |
| Revolution (S5) | 5D | ~88% | Mandate −1 |
| Varfell (S4) | 4D | ~80% | Mandate −1 |
| Hafenmark (S3) | 3D | ~73% | Mandate −1 |
| Niflhel (S2) | 2D | ~59% | Mandate −1 |

**Expected Mandate losses this Accounting:**
Crown: −0.41 expected. Niflhel: −0.41. Church and Hafenmark: −0.27 each. Revolution: −0.12. Varfell: −0.20.

Low-Stability factions (Crown, Niflhel) lose Mandate fastest under RS pressure. ✓ This mechanically accelerates political instability as the world degrades — low-Stability factions become weaker precisely when they need strength to respond. ✓

**Net RS change this Accounting:**
Operations: −3, Mend: +1, Weave partial: 0, Harvest: −0.5, Lock drift: −2, Gaps: −4 (Standard) −1 (Micro, proposed) = −5, Winter: −1. Spontaneous Gap (40% chance): −4 additional if triggers.

**Finding (P1 — SIM6-F-10): RS can drop from 18 to 3–7 in a single Accounting at Critical.** The convergence of Lock drift, Gap persistence, spontaneous Gap risk, and operation failures produces catastrophic RS loss at Critical. The seasonal cap (±10) would apply: net −10.5 is capped at −10. RS 18 → 8. Spontaneous Gap still triggers at Critical (20% → new Gap at RS 8). Next season: RS 8 → likely 0 within 1–2 seasons.

**The RS cap protects against single-Accounting Rupture but cannot prevent multi-season terminal decline at Critical.** Once RS hits Critical, the game is in a 2–4 season endgame. ✓ Design confirmed — Critical is the point of no return without dramatic intervention.

## SIMULATION 36 — C-20: OPEN P2 RESOLUTION (Mode D)

### SIM2-F-05: Witness Certainty checks for temporal displacement

**Rule proposed:** Witnesses with TS 0–29: GM discretion, no mandatory check (they rationalise). TS 30+: Certainty check Spirit TN 7 Ob 1 when a temporally displaced fact contradicts their memory. Non-practitioners with direct personal stake (the dead person's spouse, the soldier who buried the body): Certainty check regardless of TS (the relational weight makes the disjunction visceral).

Add to §9.10: "Characters other than the operating practitioner who experience direct contradiction between their memory and the new physical state may make a Certainty check (Spirit TN 7, Ob 1). Mandatory for: TS 30+ observers; characters with a named Knot to the displaced fact. Optional (GM discretion) for: TS 0–29 characters without direct relational stake."

### SIM2-F-10: Relational/Territorial/Structural Gap mechanics

**Proposed rules by scale:**

| Gap Scale | Mechanical Effect | Mending Ob |
|---|---|---|
| Relational | Affected relationship oscillates between binding and void. Treaties, bonds, and agreements involving the affected parties: +2 Ob to social operations. Cannot form new formal agreements until Mended. | Standard (5) |
| Territorial | Territory's Prosperity check each season: Ob 1 (the substrate failure disrupts material production). Navigation unreliable (+1 Ob to movement/scouting in the territory). | Entrenched (6) |
| Structural | Affected institution's authority is intermittent. Domain Actions by that institution: −1D. Its Mandate-conferring capacity suspended until Mended. | Catastrophic (7) |

Add to §9.5 or create §5.6 "Gap Scale Consequences."

### SIM2-F-11: Cross-phase opposing operations timing

**Rule proposed:** TTRPG Thread operations targeting configurations also targeted by Strategic Phase orders are flagged at Personal Phase declaration. Both are held and resolved simultaneously at Cascade Phase using §9.13 opposing procedure. The TTRPG practitioner's roll from Personal Phase stands as their input; the board game order roll is made at Cascade Phase.

Add to §7.2: "If a TTRPG Thread operation (Personal Phase) and a board game Thread order (Strategic Phase) target the same configuration with opposing intentionalities, flag at Personal Phase declaration and resolve both simultaneously at Cascade Phase using the opposing operations procedure (§9.13)."

### SIM3-F-06: Political NPC involuntary Leap social consequence

**Rule proposed (consolidated from SIM4 scenario):** Involuntary Leap in a public or social scene: Composure −1 (the momentary absence is visible). If the practitioner is in a formal context (court, negotiation, ceremony): also make a Circles check TN 7 Ob 1. Failure: a rumour is generated that the practitioner was "absent" or "distant" at a critical moment.

Add to §9.18 after existing involuntary Leap rules.

### SIM4-F-03: Knot Crisis affecting Past-Oriented Pull

**Rule proposed:** An active Knot Crisis caused by the target event adds +1 Ob to Past-Oriented Pulling targeting that event. The ongoing relational weight of the grief makes the temporal thread more firmly actualised — loved ones' suffering has woven itself into the thread's current configuration.

Add to §2.4 Past-Oriented Pulling Ob modifiers.

### SIM4-F-04: Multiple Mending successes per season stack

**Explicit statement:** Multiple successful Mending operations per season stack their RS gains. Each successful Mend: RS +1 (or +2 at Southernmost). Two practitioners each Mending successfully: RS +2 that Accounting. This is the intended path for the Revolution's Community Mending strategy and multi-practitioner expeditions.

Add to §2.6 Mending or §5.2 RS Sources table: "Multiple Mending successes in a season stack. Each successful Mend contributes its RS gain independently at Accounting."

### SIM4-F-06: Both-succeed opposing ops RS cost

**Rule proposed:** When both practitioners succeed in opposing Thread operations (§9.13 both-succeed outcome), the Shifting Object formation costs RS −1 (substrate strained by contradictory successful directions). Add to §9.13 both-succeed row.

### SIM4-F-07: Contested-operation RS spiral sidebar

**Text to add to §9.13, after the outcome table:**

> **GM sidebar — Contested Thread operations and RS:** When two practitioners oppose each other's Thread operations, the most likely outcome (~53% at comparable pools) is a Shifting Object — neither practitioner achieves their intent, both pay Coherence, and a substrate scar persists. Shifting Objects left unMended deteriorate to Gaps within 1d3 seasons. Faction conflicts that routinely involve opposing Thread operations will accelerate RS degradation faster than either faction's uncontested operations would. The game prices Thread-as-weapon use: using Thread operations as faction warfare tools is more destructive to the substrate than using them for their intended purposes.

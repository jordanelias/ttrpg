## Q4 — What is the action economy of scene combat?

**Brief:** Canonical Combat Pool (Agi×2 + Hist + 3) is split between Offence and Defence each round. Stamina is a separate resource (max = Stam derived; per-action costs in §7). The canonical action economy is **continuous pool split + discrete stamina cost**. The videogame medium permits more rigid economies (action-point caps, BP banking, stance flow) — does any of them improve on canonical?

### Candidates

| ID | Economy | Source |
|---|---|---|
| **E1** | Canonical pool split + stamina (baseline) | combat_v30 §1, §7 |
| E2 | Action-point cap (2 actions per round, XCOM-like) | XCOM 2 |
| E3 | AP/MP separated (Action / Movement points) | Wakfu, Dofus |
| E4 | Boost banking (save BP across rounds for big effect) | Octopath Traveler |
| E5 | Stance flow (each attack ends in a stance gating next attack) | Absolver, Way of the Samurai |
| E6 | Hybrid: canonical pool + Stamina-banking for Heavy attack | E1 + E4 selectively |
| E7 | Posture meter (Stamina ends fight on break) | Sekiro |

### Compact NERS-all-directions per candidate

| Candidate | Top-down | Bottom-up | Vertical | Diagonal | Lateral | Horizontal | Verdict |
|---|---|---|---|---|---|---|---|
| **E1 canonical** | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | All clean. Status quo. |
| E2 action-point cap | ⚠ | ✗ | ⚠ | ⚠ | ⚠ | ✓ | **Forks canonical math** — pool split is more granular than 2-action cap. Reject. |
| E3 AP/MP split | ⚠ | ⚠ | ⚠ | ⚠ | ⚠ | ✓ | Stamina already does Movement-Point work canonically; explicit split over-discretizes. Reject. |
| E4 boost banking | ⚠ | ⚠ | ⚠ | ⚠ | ⚠ | ✓ | Banking adds resource layer above canonical pool — forks math. Reject as standalone. |
| E5 stance flow | ⚠ | ⚠ | — | ⚠ | — | ⚠ | Cool for C duel only. Doesn't generalize to A or B. **C-only inspiration.** |
| **E6 hybrid (canonical + Stamina-banking)** | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | Stamina-banking for Heavy attack is canon-adjacent (§7 Heavy already costs 8 Stamina) — formalizing as a pre-commit choice adds depth without forking math. Adopt. |
| E7 posture (Sekiro) | ⚠ | ⚠ | ⚠ | ⚠ | — | ✓ | **C only** — duel-ender via Stamina is dramatic but doesn't generalize. Adopt for C. |

### Aggregate read on Q4

**Canonical pool split + Stamina is the right economy.** No primary replacement is justified.

**Two adoptions on top of baseline:**
1. **E6 — Stamina-banking for Heavy attack.** Player can save Stamina across rounds (cap 1.5× normal max) to commit a Heavy attack. Canon-adjacent — §7 already gates Heavy at 8 Stamina; surfacing it as a deliberate banking choice adds tactical depth. Across A/B/C.
2. **E7 — Posture as duel-ender** in **C only**. When Stamina depletes in a duel, opponent yields rather than dies (or dies if `to-the-death` wager set). Aligns with canonical Out of Breath (§7 −2D) but escalates it to a fight-ender at duel scale where the camera and stakes support it.

### Net change to canon

| File | Change |
|---|---|
| combat_v30 §7 | Add: "Stamina banking — player may delay Take a Breath, accumulating up to 1.5× max Stamina, spent on Heavy attack pre-declaration." |
| combat_v30 §C (new — duel chapter) | Add: "Duel posture rule — at C-architecture scale, Stamina depletion yields the duel rather than penalizing actions." |

## Q5 — When duel vs when group? Trigger logic.

**Brief:** Chunks 1–2 establish **A (map+move), B (slot formation), C (duel)** as three architectures. Chunk 4 favors S7-hybrid which collapses A and B into the same spatial substrate (continuous within a zone). C remains specialized. **The remaining question is: when does the engine fire which architecture?**

### Architecture decision: do we need B at all?

If S7-hybrid (Chunk 4) gives every scene continuous-position-within-a-zone capability, **B (DD slot formation) is no longer required as a distinct architecture**. Routine encounters can run on the same A substrate at faster tempo (smaller zone, fewer actors, simpler Stunt vocabulary).

| Direction | Two-architecture (A + C) | Three-architecture (A + B + C) | Notes |
|---|---|---|---|
| Top-down | ✓ | ⚠ | Two systems is simpler design surface |
| Bottom-up | ✓ | ✓ | Both compose canonical math |
| Vertical | ✓ | ⚠ | B doesn't extend to mass — extra mode-switch in three-arch |
| Diagonal | ✓ | ⚠ | A+C share substrate via S7-hybrid; B is an island |
| Lateral | ✓ | ⚠ | Three-arch needs three combat UI languages |
| Horizontal | ✓ | ⚠ | Player learns one combat idiom (A), one set-piece (C) |
| **Implementation cost** | Lower | Higher | Two systems > three |

**Verdict:** **Drop B as a distinct architecture.** Use A (S7-hybrid) for all general combat — adjust *zone size, encounter density, animation speed* to span the use cases B would have served (corridor ambush = small zone with 4 actors; field battle = large zone with 12 actors). Use C only for set-piece duels.

This is a meaningful simplification. The DD slot pattern was attractive for routine combat, but **S7-hybrid + A makes that pattern unnecessary** — a small zone with fixed actor positions and adjacency-only reach IS the slot formation, just rendered on the same substrate. The slot abstraction was a TTRPG/board-game cognitive crutch; the videogame doesn't need it.

### C trigger logic

Given two architectures (A + C), when does C fire?

#### Triggers — what conditions invoke C?

| Trigger | Description | Example |
|---|---|---|
| **T-Wager** | Both parties accept a duel wager (Conviction stake, Renown stake, item stake, life stake) | Solmund challenges a Crown duelist over a Reputation slight |
| **T-Cultural** | Cultural code mandates 1v1 (Varfell single combat, Crown formal duel, Practitioner duel) | Varfell warband leader confronts player |
| **T-Boss** | Authored narrative beat — climactic 1v1 confrontation | Final-act antagonist duel |
| **T-Thread** | Practitioner-vs-Practitioner Threadwork duel — both Leap, witnesses watch | Two Practitioners on a hill, audience below |
| **T-Hero-Officer** | Player isolates an enemy officer mid-mass-battle (via I2 continuous-zoom or I4 parallel) | Hunting the enemy general at battle's climax |
| **T-Honor-call** | NPC issues honor-bound challenge that player can accept or refuse | "Meet me at dawn at the chapel" |

#### Trigger logic

```
IF participants == 2 AND
   (T-Wager OR T-Cultural OR T-Boss OR T-Thread OR T-Hero-Officer OR T-Honor-call):
    → invoke C (duel architecture, Pirates!-stance + Sekiro-posture)
ELSE:
    → invoke A (S7-hybrid, scene combat)

C can also be REJECTED by either party:
- Player can refuse a T-Cultural challenge (Reputation cost, Conviction cost — canonical §13.2)
- NPC can refuse a player's wager (per AI temperament)
- C fallback: A engages instead, but with a Reputation/Conviction tag for "ducked the duel"
```

#### NERS-all-directions on this trigger logic

| Direction | N | E | R | S | Notes |
|---|---|---|---|---|---|
| Top-down | ✓ | ✓ | ✓ | ✓ | Clear when each fires; player understands the genre signal |
| Bottom-up | ✓ | ✓ | ✓ | ✓ | Triggers compose from canonical primitives (Reputation, Conviction, faction culture, narrative tags) |
| Vertical | ✓ | ✓ | ✓ | ✓ | T-Hero-Officer is the natural mass-scale → C bridge; canonical §D.2 alignment |
| Diagonal | ✓ | ✓ | ✓ | ✓ | T-Thread bridges combat ↔ thread; T-Wager bridges combat ↔ social contest |
| Lateral | ✓ | ✓ | ✓ | ✓ | Wager mechanic shares with social contest stakes spec |
| Horizontal | ✓ | ✓ | ✓ | ✓ | Player learns "duels are special" early; expectations stable |

**Verdict:** Trigger logic clean. **C fires on six discriminable conditions, all canon-adjacent. Default is A.**

### Net change to canon

| File | Change |
|---|---|
| combat_v30 (new §) | Add: "Architecture selector — C (duel) fires on 2-participant triggers (Wager / Cultural / Boss / Thread / Hero-Officer / Honor-call). Default is A (general scene combat)." |
| combat_v30 §B (new — duel chapter) | Net-new: full duel architecture spec — stance UI, Pirates!-style commit-and-tell, posture-as-fight-ender, audience for Reputation. |
| combat_v30 §11.5 (existing) | Extend: T-Cultural-refusal Reputation/Conviction tag rules. |

## Q6 — How does combat ↔ fieldwork bridge?

**Brief:** Canonical combat_v30 §11.5 specifies fieldwork→combat (Exposure quiet/conspicuous/public) and combat→fieldwork (post-battle scene at battle site, fled-site degrades over seasons). With the videogame-medium continuous spatial substrate (Chunk 4 S7-hybrid) and continuous-zoom interface (Chunk 5 I2), the bridge becomes more than a transition — it becomes a **non-transition**: scene combat happens *inside* the fieldwork map, not as a separate scene.

### Bridge candidates

| ID | Pattern | Description |
|---|---|---|
| **F1** | Modal transition (canonical) | Fieldwork scene → "combat begins" screen → combat scene → "combat ends" → fieldwork resumes |
| **F2** | Same-map continuous (videogame-native) | Fieldwork is happening on the map; combat erupts on the same map; pacing changes (real-time → IP-gauge), camera focuses on combatants, but no scene-switch |
| F3 | Stealth-graduated (Mutant Year Zero) | Pre-combat stealth phase explicit; player can take silent eliminations before combat triggers; Exposure rises with each detected action |
| F4 | Encounter-node (DD model) | Fieldwork is a node graph; combat is a fixed encounter inside a node; spatial continuity broken |

### Compact NERS

| Bridge | Top-down | Bottom-up | Vertical | Diagonal | Lateral | Horizontal | Verdict |
|---|---|---|---|---|---|---|---|
| F1 modal (canonical baseline) | ⚠ | ✓ | ✓ | ⚠ | ✓ | ⚠ | Canonical baseline; immersion tax |
| **F2 same-map continuous** | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | **Highest fit with S7-hybrid + I2.** Adopt. |
| F3 stealth-graduated | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ | Layer atop F2. Adopt. |
| F4 encounter-node | ⚠ | ✓ | ⚠ | ✗ | ⚠ | ⚠ | Breaks spatial continuity Chunk 4 just established. Reject. |

### Adopted bridge: F2 + F3

**F2 — same-map continuous** as base. Combat erupts on the existing fieldwork map. Camera tightens on combatants. Time-shape shifts from continuous-fieldwork-pace to canonical IP-gauge. Pause/resume per scene combat conventions.

**F3 — stealth-graduated** as the *trigger ramp*. Before combat begins, the player has a pre-combat surface where:
- Detection level rises with player visibility (canonical Exposure §11.5: quiet/conspicuous/public)
- Silent eliminations possible at quiet level (single-action, instant resolution if undetected and unopposed)
- First detection escalates Exposure tier; combat begins at conspicuous when any actor commits hostile action visible to non-hostile witnesses
- Mass-battle scenes start at public Exposure by definition

This handles the canonical Exposure ladder mechanically rather than narratively.

### Specific cross-system flows under F2+F3

#### Investigation-into-combat
Player enters a hostile camp under quiet Exposure. Picks off two sentries silently (stealth eliminations). On third, alarm raises — Exposure jumps to conspicuous. Remaining enemies wake; combat begins on the same map without a scene-transition. Bodies of stealth kills are still on the map and contribute to post-combat investigation.

#### Combat-into-investigation
Combat resolves. Bodies, dropped equipment, tracks remain on the map. Player can immediately investigate without a transition. Site persists per canonical §11.5 — degrades over seasons if abandoned.

#### Combat-then-flee
Player flees combat. Enemies pursue (pursuit-as-scene per Chunk 5 — possibly C if isolated, A if group). Site canonically degrades over seasons; if revisited, may have been looted, inhabited, or naturalized.

#### Cross-faction interruption
Faction A and B are fighting in a zone. Player enters during combat. Player is outside Exposure circle of either side initially. Player can intervene (joining one side), withdraw, or witness — canonical §11.5 witnesses for Reputation Cascade (§13.2) apply to player too.

These flows are all enabled by F2+F3 and most are **awkward or impossible under F1** (canonical modal transition).

### Net change to canon

| File | Change |
|---|---|
| combat_v30 §11.5 | **Major rewrite.** Replace modal transition language with same-map-continuous bridge spec. Stealth-graduated trigger ramp added. Map persistence for site state retained. |
| fieldwork_v30 (existing — provincial layer) | Cross-link: combat is one of fieldwork's possible scene-states, not a separate system. |

## Composite cross-cutting decisions from Chunks 3–6

The decisions accumulate into a coherent stack:

| Decision | Source | Impact |
|---|---|---|
| **T2+T4 time-shape unified across scales** | Chunk 3 | Same clock for scene + mass; eliminates phase-mode-switch (§A.7) |
| **S7-hybrid spatial substrate** | Chunk 4 | World zones unchanged; sub-zone continuous at scene scale; mass canon zero rework |
| **I2 continuous-zoom interface (with I1 as MVP)** | Chunk 5 | Three-mode `mass_battle_v30` collapses to videogame-medium framing |
| **B architecture dropped; A+C only** | Chunk 6 Q5 | Two combat architectures, not three; simpler design surface |
| **C trigger logic — 6 discriminable conditions** | Chunk 6 Q5 | Duel firing predictable and canon-adjacent |
| **E1 + E6 + E7 action economy** | Chunk 6 Q4 | Canonical pool split + Stamina-banking for Heavy + posture-as-duel-ender |
| **F2+F3 fieldwork bridge** | Chunk 6 Q6 | Combat is a state of the map, not a separate scene |

These compose: same time-shape + same spatial substrate + same map = **scene combat is a tempo + camera shift on a continuous fieldwork-mass map, with C as a specialized cinematic mode for 1v1 narrative beats**.

The TTRPG zone abstraction served as the cognitive scaffolding that made canonical mechanics tractable to author. The videogame medium replaces the scaffolding with a continuous spatial substrate that reaffirms the same canonical primitives at each scale (zones for mass, sub-zone continuous for scene, sub-zone-with-stance-UI for duel) **without any of them being modal transitions**.
